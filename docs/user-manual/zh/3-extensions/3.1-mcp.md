# 3.1 MCP 服务器管理

## 概览

YKW Bridge 当前只管理 **Claude Code** 的 MCP 服务器。

MCP 配置统一保存在 YKW Bridge 的数据库中，并在启用后同步到 Claude 的 `~/.claude.json` 文件里的 `mcpServers` 节点。当前界面只保留 Claude 工作流的 MCP 开关。

## 打开 MCP 面板

点击顶部导航栏的 **MCP** 按钮。

## 添加 MCP 服务器

### 使用预设模板

1. 点击右上角 **+**
2. 从“预设”下拉框选择模板
3. 按需修改命令、参数或 URL
4. 点击“保存”

常见预设包括：

- `fetch`
- `time`
- `memory`
- `sequential-thinking`
- `context7`

### 自定义配置

自定义模式下常用字段如下：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| Server ID | 是 | 唯一标识 |
| Name | 否 | 显示名称 |
| Description | 否 | 描述信息 |
| Transport Type | 是 | `stdio` / `http` / `sse` |
| Command | `stdio` 时必填 | 启动命令 |
| Arguments | 否 | 命令参数 |
| URL | `http/sse` 时必填 | 远程服务地址 |
| Headers | 否 | HTTP 请求头 |
| Environment Variables | 否 | 传给 MCP 进程的环境变量 |

## 启用与同步

当前 MCP 面板只保留 **Claude** 开关。

启用后会发生：

1. 数据库记录该服务器已对 Claude 启用
2. 配置写入 `~/.claude.json`
3. Claude Code 下次启动或重载配置时即可加载

禁用后会：

1. 更新数据库状态
2. 从 `~/.claude.json` 中移除对应服务器

## 导入现有配置

如果你已经在 Claude 本地配置过 MCP 服务器，可以点击 **导入**：

1. 读取现有 `~/.claude.json`
2. 把其中的 `mcpServers` 导入到 YKW Bridge
3. 后续改动统一在 YKW Bridge 中管理

## 配置示例

### `stdio`

```json
{
  "command": "uvx",
  "args": ["mcp-server-fetch"]
}
```

### `http`

```json
{
  "url": "http://localhost:8080/mcp"
}
```

### Claude MCP 文件示例

```json
{
  "mcpServers": {
    "mcp-fetch": {
      "command": "uvx",
      "args": ["mcp-server-fetch"]
    }
  }
}
```

## Deep Link 导入

MCP 类型的 Deep Link 当前只支持：

- `resource=mcp`
- `apps=claude`

如果链接里仍然带有旧版多应用标识，导入会失败。

## 注意事项

- `stdio` 模式下，命令必须已经安装并位于 PATH 中
- 修改 MCP 后，建议重新打开 Claude Code 或开始新会话
- 如果本机不存在 Claude 配置路径，启用开关不会写入 live 文件，但数据库配置仍会保留

## 常见问题

### 服务器启动失败

优先检查：

- 命令是否已安装
- 参数是否正确
- 本地环境变量是否完整

### 配置没有生效

优先检查：

- Claude 开关是否已启用
- `~/.claude.json` 是否被其他工具覆盖
- Claude Code 是否已经重新加载配置
