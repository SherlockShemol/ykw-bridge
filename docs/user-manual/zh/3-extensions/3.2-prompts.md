# 3.2 Prompts 管理

## 概览

Prompts 用于管理系统提示词预设。YKW Bridge 当前支持为以下目标维护独立提示词：

- **Claude Code**
- **Claude Desktop**

两个目标的预设彼此独立。

## 打开面板

点击顶部导航栏的 **Prompts** 按钮。

## 创建预设

1. 点击右上角 **+**
2. 输入预设名称
3. 在 Markdown 编辑器中写入提示词
4. 点击“保存”

编辑器支持：

- Markdown 语法高亮
- 实时预览
- 常用格式快捷按钮

## 激活预设

点击预设卡片上的开关即可启用。

同一目标下始终只有一个预设处于激活状态；启用新的预设时，旧的会自动停用。

## 同步目标

激活后，YKW Bridge 会把内容写回对应 live 文件：

| 目标 | 同步文件 |
| --- | --- |
| Claude Code | `~/.claude/CLAUDE.md` |
| Claude Desktop | Claude Desktop profile 目录中的 `CLAUDE.md` |

> Claude Desktop 的 profile 路径会受到系统平台和设置页自定义目录影响；以应用内“状态/Doctor”显示为准。

## 编辑与删除

### 编辑

1. 点击“编辑”
2. 修改名称或内容
3. 点击“保存”

如果编辑的是当前激活预设，保存后会立即同步到 live 文件。

### 删除

1. 点击“删除”
2. 确认操作

当前激活的预设不能直接删除，需要先停用或切换到其他预设。

## 智能回填

YKW Bridge 会在切换预设前先读取当前 live 文件，并把你在外部手动改过的内容回填到当前预设，避免被覆盖。

典型触发时机：

- 切换到另一个预设时
- 编辑当前激活预设时
- 首次启动时导入现有 live 文件

## Deep Link

Prompt Deep Link 当前支持：

- `app=claude`
- `app=claude_desktop`

旧版多应用标识已不再支持。

## 使用建议

- Claude Code 和 Claude Desktop 如需相同提示词，请分别创建对应预设
- 经常在外部手改 `CLAUDE.md` 时，切换前可以先回到 YKW Bridge 看一下是否已回填
- 对长期稳定的团队提示词，建议配合导出或 Deep Link 一起管理
