# 5.3 深度链接协议

## 功能说明

YKW Bridge 当前使用 `ykwbridge://` 深度链接协议，可以通过链接一键导入供应商、Prompt、MCP 和 Skill 仓库配置。

**常见场景**：

- 团队共享供应商或 MCP 配置
- 在教程里提供一键导入入口
- 在不同设备间快速复用同一套配置

## 协议格式

### V1 协议

```text
ykwbridge://v1/import?resource={type}&...
```

### 通用参数

| 参数 | 必填 | 说明 |
|------|------|------|
| `resource` | 是 | 资源类型：`provider` / `prompt` / `mcp` / `skill` |

## 各资源类型参数

### 供应商（resource=provider）

| 参数 | 必填 | 说明 |
|------|------|------|
| `app` | 是 | `claude` / `claude_desktop` |
| `name` | 是 | 供应商名称 |
| `endpoint` | 否 | API 地址，支持逗号分隔多个 URL |
| `apiKey` | 否 | API 密钥 |
| `homepage` | 否 | 官网地址 |
| `model` | 否 | 默认模型 |
| `haikuModel` | 否 | Claude Haiku 默认模型 |
| `sonnetModel` | 否 | Claude Sonnet 默认模型 |
| `opusModel` | 否 | Claude Opus 默认模型 |
| `notes` | 否 | 备注 |
| `icon` | 否 | 图标名 |
| `config` | 否 | Base64 编码的内联配置 |
| `configFormat` | 否 | `json` / `toml` |
| `enabled` | 否 | 导入后是否立即启用 |
| `usageScript` | 否 | Base64 编码的用量查询脚本 |
| `usageEnabled` | 否 | 是否启用用量查询 |
| `usageApiKey` | 否 | 用量查询专用 API Key |
| `usageBaseUrl` | 否 | 用量查询专用地址 |
| `usageAccessToken` | 否 | 用量查询专用访问令牌 |
| `usageUserId` | 否 | 用量查询专用用户 ID |
| `usageAutoInterval` | 否 | 自动查询间隔（分钟） |

说明：

- `endpoint`、`apiKey`、`homepage` 可以直接写在 URL 里
- 也可以通过 `config` 传一份 Base64 内联配置，让系统自动回填
- 当前文档不再把远程 `configUrl` 作为推荐用法

### Prompt（resource=prompt）

| 参数 | 必填 | 说明 |
|------|------|------|
| `app` | 是 | `claude` / `claude_desktop` |
| `name` | 是 | Prompt 名称 |
| `content` | 是 | Base64 编码后的 Prompt 内容 |
| `description` | 否 | 描述 |
| `enabled` | 否 | 导入后是否立即启用 |

### MCP（resource=mcp）

| 参数 | 必填 | 说明 |
|------|------|------|
| `apps` | 是 | 当前仅支持 `claude` |
| `config` | 是 | Base64 编码后的 JSON，内容需包含 `mcpServers` |
| `enabled` | 否 | 是否启用 |

### Skill 仓库（resource=skill）

| 参数 | 必填 | 说明 |
|------|------|------|
| `repo` | 是 | 仓库，格式 `owner/name` |
| `directory` | 否 | 仓库中的目录路径 |
| `branch` | 否 | 分支名 |

Skill Deep Link 不需要单独传 `app`；当前会按 Claude Skills 仓库导入。

## 导入示例

### 导入 Claude 供应商

```text
ykwbridge://v1/import?resource=provider&app=claude&name=My%20Provider&endpoint=https%3A%2F%2Fapi.example.com&apiKey=sk-xxx
```

### 导入 Claude Desktop 供应商

```text
ykwbridge://v1/import?resource=provider&app=claude_desktop&name=Desktop%20Provider&endpoint=https%3A%2F%2Fapi.example.com&apiKey=sk-xxx
```

### 导入 MCP

```text
ykwbridge://v1/import?resource=mcp&apps=claude&config=BASE64_ENCODED_JSON
```

### 导入 Prompt

```text
ykwbridge://v1/import?resource=prompt&app=claude&name=Code%20Review&content=BASE64_ENCODED_CONTENT
```

### 导入 Skill 仓库

```text
ykwbridge://v1/import?resource=skill&repo=owner/repo&directory=skills/my-skill&branch=main
```

## 如何生成链接

### 手动生成

1. 准备好参数
2. 按 `ykwbridge://v1/import?...` 形式拼接
3. 对特殊字符做 URL 编码
4. 对 `content` / `config` / `usageScript` 这类字段先做 Base64 编码

### JavaScript 示例

```javascript
const params = new URLSearchParams({
  resource: "provider",
  app: "claude",
  name: "My Provider",
  endpoint: "https://api.example.com",
  apiKey: "sk-xxx",
});

const url = `ykwbridge://v1/import?${params.toString()}`;
```

### Base64 编码示例

```javascript
const promptContent = Buffer.from("# Role\nYou are a reviewer", "utf8").toString("base64");
```

## 使用深度链接

点击链接后，YKW Bridge 会：

1. 校验协议与参数格式
2. 打开导入确认对话框
3. 展示配置预览
4. 等你确认后再执行导入

**安全提示**：只导入来自可信来源的链接。

## 协议注册

### 自动注册

YKW Bridge 安装时会自动注册 `ykwbridge://` 协议。

### 手动检查

如果协议没有生效：

- macOS：重新安装应用，或重新执行应用注册流程
- Windows：检查 `HKEY_CLASSES_ROOT\ykwbridge`
- Linux：检查 `.desktop` 文件中的协议声明

## 安全建议

深度链接里可能包含敏感信息，比如：

- API Key
- 用量查询脚本
- 第三方端点地址

建议：

1. 不要公开分享带真实密钥的链接
2. 分享前先替换敏感字段
3. 导入前先看确认对话框里的预览

## 故障排除

### 链接无法打开

优先检查：

1. YKW Bridge 是否已安装
2. `ykwbridge://` 是否已注册
3. 链接格式是否正确

### 导入失败

**常见原因**：

- Base64 编码错误
- JSON / TOML 内容本身有误
- 缺少必填字段
- `app` 或 `apps` 写成了已不支持的旧值

**解决方法**：

1. 重新检查原始内容
2. 重新做 Base64 编码
3. 确认 `provider` / `prompt` 只使用 `claude` 或 `claude_desktop`
4. 确认 `mcp.apps` 只使用 `claude`
