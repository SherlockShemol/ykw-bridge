# 5.1 配置文件说明

## YKW Bridge 数据目录

### 默认位置

当前默认数据目录：`~/.ykw-bridge/`

除非你在设置里覆盖了某些目录，否则数据库、设备级设置、Skills、Workspace、备份和 Claude Desktop 托管 Profile 都会放在这里。

### 常见目录结构

```text
~/.ykw-bridge/
├── ykw-bridge.db              # SQLite 主数据库
├── settings.json              # 设备级设置
├── config.json                # 聚合配置文件
├── backups/                   # 数据库/导入/环境变量删除前备份
├── skills/                    # Skills 单一事实源（SSOT）
├── skill-backups/             # Skills 卸载备份
├── workspace/                 # Workspace 文件
│   └── memory/                # Daily Memory
├── claude-desktop/
│   └── profile/               # Claude Desktop 托管 Profile（默认）
└── crash.log                  # 崩溃日志（按需生成）
```

### 数据库存什么

`ykw-bridge.db` 是当前版本的主数据库，主要保存：

- 供应商配置与自定义端点
- MCP 服务器
- Prompt 预设
- Skills 安装状态与仓库信息
- 代理、故障转移、请求日志
- 模型定价与健康状态

### `settings.json`

`settings.json` 存的是设备级设置，不跟数据库一起跨设备同步。典型字段类似：

```json
{
  "visibleApps": {
    "claude": true,
    "claudeDesktop": true
  },
  "claudeConfigDir": null,
  "claudeDesktopAppPath": null,
  "claudeDesktopProfileDir": null,
  "launchOnStartup": false
}
```

### `backups/`

`backups/` 里会出现几类备份文件：

- 导入配置前自动创建的备份
- 数据库恢复前的安全备份
- 删除环境变量冲突前生成的 `env-backup-*.json`

## Claude Code 配置

### 默认目录

默认路径：`~/.claude/`

### 常见文件

```text
~/.claude/
├── settings.json     # Claude Code 主配置
├── CLAUDE.md         # Prompt / 系统说明
└── skills/           # 同步后的 Skills
```

### MCP 配置文件

Claude Code 的 MCP 默认写到：`~/.claude.json`

如果你在设置里覆盖了 Claude Code 配置目录，MCP 文件会跟着改到覆盖目录同级的 `*.json` 侧车文件，而不是继续固定写在默认位置。

## Claude Desktop 托管 Profile

### 默认目录

默认路径：`~/.ykw-bridge/claude-desktop/profile/`

也可以在设置中改成自定义目录。

### 这个目录里通常有什么

托管 Profile 会维护 Claude Desktop 工作流需要的文件，例如：

- 受管配置
- `CLAUDE.md`
- `skills/`
- 本地网关 / 证书相关文件
- Claude Desktop 本地会话目录

这里描述的是 Claude Desktop 工作流的托管目录，不是某个独立第三方应用的配置目录。

## Workspace 与 Daily Memory

默认路径：

- `~/.ykw-bridge/workspace/`
- `~/.ykw-bridge/workspace/memory/`

Workspace 面板里的常用文件和 Daily Memory 会保存在这里。

## 配置优先级

YKW Bridge 的当前工作方式可以理解成三层：

1. **数据库**：主数据源
2. **Claude / Claude Desktop 实际配置文件**：切换供应商、Prompt、MCP、Skills 时写入
3. **回填读取**：编辑当前配置时，会尽量从现有 live 文件读取内容并回填到 UI

## 手动编辑建议

### 可以手动编辑

- Claude Code 的配置文件
- Claude Desktop 托管 Profile 中你明确知道用途的文本文件
- YKW Bridge 的 `settings.json`

### 不建议手动编辑

- `ykw-bridge.db`
- `backups/` 里的备份文件
- 不了解用途的 Claude Desktop 托管网关文件

### 编辑后如何同步

如果你手动修改了 Claude 相关 live 配置：

1. 打开 YKW Bridge
2. 进入对应供应商或配置面板
3. 确认回填结果是否符合预期
4. 再保存一次，让数据库与 live 文件重新对齐

## 配置迁移

### 旧版 `.cc-switch`

旧版本可能用的是 `~/.cc-switch/`。当前默认目录已经切换到 `~/.ykw-bridge/`。

如果新目录不存在，但旧目录存在，YKW Bridge 可能会自动导入旧数据做兼容迁移。所以现在的判断方式是：

- `~/.cc-switch/`：历史兼容来源
- `~/.ykw-bridge/`：当前默认目录

### 跨设备迁移

可以用三种方式迁移：

1. 导出 / 导入配置
2. 用 WebDAV 同步
3. 手动备份 `~/.ykw-bridge/` 下需要的内容后再恢复

## 备份建议

### 定期导出

建议定期从设置里的数据管理面板导出一份配置，尤其是在：

- 批量调整供应商前
- 做大规模 Prompt / MCP / Skills 变更前
- 切换设备前

### 导出通常包含什么

- 供应商配置
- MCP 服务器
- Prompt 预设
- Skills 元数据
- 可同步的应用数据

### 不建议直接依赖什么

不建议把运行期日志、临时状态文件或设备级路径覆盖设置当成跨设备恢复的唯一依据。
