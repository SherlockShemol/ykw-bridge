# 5.2 常见问题 FAQ

## 安装问题

### macOS 安装

YKW Bridge 的 macOS 版本已做代码签名和公证，正常情况下可以直接安装。若遇到异常，优先从 [Releases 页面](https://github.com/SherlockShemol/ykw-bridge/releases) 重新下载安装包。

### Windows 安装后无法启动

**常见原因**：

- 缺少 WebView2 运行时
- 杀毒软件拦截

**解决方法**：

1. 安装 [Microsoft Edge WebView2](https://developer.microsoft.com/en-us/microsoft-edge/webview2/)
2. 将 YKW Bridge 加入杀毒软件白名单

### Linux 启动报错

**问题**：AppImage 无法启动

**解决方法**：

```bash
chmod +x YKW-Bridge-*.AppImage
./YKW-Bridge-*.AppImage --no-sandbox
```

## 供应商问题

### 切换供应商后不生效

**原因**：不同工作流读取配置的时机不同。

**解决方法**：

- Claude Code 普通模式：关闭并重新打开终端，或重启 IDE 中的 Claude Code 进程
- Claude Code 接管 / 代理模式：大多数情况下可以直接热切换
- Claude Desktop 托管 Profile：如果当前实例还持有旧配置，退出 Claude.app 后重新打开一次

### API Key 无效

**检查步骤**：

1. 确认复制时没有多余空格或换行
2. 确认 Key 没有过期或被吊销
3. 确认端点地址与认证方式匹配
4. 用速度测试或流式测试验证实际连通性

### 如何恢复官方登录

**操作步骤**：

1. 选择 Claude 的「官方登录」预设
2. 点击「启用」
3. 重新打开对应的 Claude 工作流
4. 按官方流程完成登录

> `codex_oauth` 仍然保留，但它是一个可挂在 Claude 工作流里的供应商路径，不是单独的 Codex 应用。

## 代理问题

### 代理服务启动失败

**常见原因**：端口被占用

**解决方法**：

1. 检查端口占用：

   ```bash
   # macOS/Linux
   lsof -i :49152

   # Windows
   netstat -ano | findstr :49152
   ```

2. 关闭占用端口的程序
3. 或到「设置 → 代理服务」里恢复默认端口

### 代理模式下请求超时

**常见原因**：

- 网络不稳定
- 上游供应商异常
- 代理配置有误

**解决方法**：

1. 先确认本地网络可用
2. 暂时关闭代理，直接访问供应商端点做对照
3. 检查供应商卡片中的端点、认证和模型配置

### 关闭代理后配置未恢复

**常见原因**：代理异常退出，未完成恢复流程。

**解决方法**：

1. 打开当前供应商
2. 检查端点与认证字段
3. 重新保存一次，让 live 配置回到当前数据库状态

## 故障转移问题

### 故障转移没有触发

**检查清单**：

- [ ] 代理服务正在运行
- [ ] 应用接管已开启
- [ ] 自动故障转移已开启
- [ ] 队列里确实有备用供应商

### 频繁触发故障转移

**常见原因**：

- 主供应商本身不稳定
- 熔断阈值过低

**解决方法**：

1. 先检查主供应商状态
2. 适当调高失败阈值
3. 必要时更换主供应商

### 所有供应商都熔断了

**解决方法**：

1. 等待熔断时间结束
2. 或重启代理服务重置状态

## 数据问题

### 配置丢失

**常见原因**：

- 数据目录被误删
- 数据库损坏
- 误以为还在使用旧版 `.cc-switch`

**解决方法**：

1. 检查 `~/.ykw-bridge/` 是否存在
2. 查看 `~/.ykw-bridge/backups/` 是否有可恢复备份
3. 必要时从导出的配置文件重新导入

### 导入配置失败

**常见原因**：

- 文件格式不正确
- 内容不完整
- 数据版本不兼容

**解决方法**：

1. 确认文件是 YKW Bridge 导出的配置
2. 用文本编辑器检查 JSON 是否完整
3. 重新导出或换一份更接近当前版本的备份再试

### 用量统计数据为空

**检查清单**：

- [ ] 代理服务是否运行
- [ ] 应用接管是否开启
- [ ] 日志记录是否开启
- [ ] 是否真的有请求经过当前代理链路

## 配额与余额

### 为什么有的供应商自动显示配额，有的需要手动启用？

当前中文手册保留的自动查询范围是：

- Claude 官方登录
- GitHub Copilot
- Codex OAuth 反向代理

其他供应商（包括 Token Plan、第三方余额查询、各类中转服务）都需要到供应商卡片的「用量查询」面板里手动启用。原因很简单：同一个请求地址可能既支持套餐配额，也支持账户余额，YKW Bridge 不会替你猜。

### 官方订阅类供应商没有显示配额

**检查步骤**：

1. 确认供应商处于当前启用状态
2. 对于 Copilot / Codex OAuth，检查 OAuth Token 是否仍有效
3. 如果卡片提示「会话已过期」，到 **OAuth 认证中心**重新登录
4. 点击卡片上的刷新按钮手动重查一次

### 手动启用后仍不显示

**检查步骤**：

1. 确认「启用用量查询」开关已经打开
2. 确认已选择并保存正确模板
3. 使用「测试脚本」查看具体报错
4. 确认该供应商当前处于激活状态

## Codex OAuth 反向代理

### 启用 Codex OAuth 反向代理有什么风险？

Codex OAuth 反向代理通过逆向工程的 OAuth 流程访问 ChatGPT 账号上的 Codex 服务，可能违反 OpenAI 的服务条款，也可能带来账号受限、接口变化或长期不可用的风险。启用前请自己评估。

完整免责声明参见 [2.1 添加供应商 → Codex OAuth 反向代理](../2-providers/2.1-add.md#codex-oauth-反向代理claude-供应商)。

### 如何登录 Codex OAuth？

完整的 Device Code 登录流程、两个入口（添加供应商面板 / OAuth 认证中心）、多账号管理和常见失败场景，参见 [2.1 添加供应商 → Codex OAuth 反向代理（Claude 供应商）](../2-providers/2.1-add.md#codex-oauth-反向代理claude-供应商)。

### Codex OAuth 登录后配额没显示

**解决方法**：

1. 确认已在 **OAuth 认证中心**完成登录
2. 检查 Token 是否仍在有效期内
3. 如果卡片显示「会话已过期」，移除该账号后重新登录

## 其他问题

### 托盘图标不显示

**macOS**：

- 检查系统设置中的菜单栏图标显示

**Windows**：

- 检查任务栏图标是否被折叠隐藏

**Linux**：

- 确认系统已安装托盘支持组件，例如 `libappindicator`

### 界面显示异常

**解决方法**：

1. 切换主题后重试
2. 重启应用
3. 删除 `~/.ykw-bridge/settings.json`，让设备级设置回到默认值

### 更新失败

**解决方法**：

1. 检查网络连接
2. 从 Releases 页面重新下载安装最新版本

## 轻量模式

### 如何进入轻量模式？

从系统托盘菜单切换「轻量模式」。主窗口会关闭，但 YKW Bridge 继续驻留托盘运行。再次切换，或点击「打开主界面」即可退出。

### 轻量模式下内存会更低吗？

会。轻量模式会销毁主窗口与 WebView，通常能明显减少内存占用，同时保留托盘功能。

### 轻量模式下深链接还能唤起主界面吗？

可以。点击 `ykwbridge://` 链接时，YKW Bridge 会按需重建主窗口并弹出导入确认对话框。第一次唤起可能略慢，后续会恢复正常。

## 获取帮助

### 提交 Issue

如果以上方法都无法解决问题：

1. 访问 [GitHub Issues](https://github.com/SherlockShemol/ykw-bridge/issues)
2. 搜索是否已有类似问题
3. 如果没有，再新建 Issue
4. 附上以下信息：
   - 操作系统与版本
   - YKW Bridge 版本
   - 复现步骤
   - 相关报错

### 附带哪些诊断信息更有帮助

建议一并提供与问题直接相关的信息，例如：

- `~/.ykw-bridge/` 下的崩溃日志或备份文件
- 供应商卡片报错截图
- 代理或流式测试的错误信息
