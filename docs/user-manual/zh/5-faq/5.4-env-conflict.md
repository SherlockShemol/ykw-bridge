# 5.4 环境变量冲突

## 功能说明

YKW Bridge 会自动检测系统环境变量与当前 Claude 工作流配置的冲突，避免配置被意外覆盖。

当前这套检查只面向 `Claude` / `Claude Desktop`，文档里不再保留旧版多应用环境变量说明。

**当前会扫描的变量范围**：

- `ANTHROPIC_API_KEY`
- `ANTHROPIC_AUTH_TOKEN`
- `ANTHROPIC_BASE_URL`
- 以及其他名称中包含 `ANTHROPIC` 的变量

## 冲突警告

当检测到冲突时，界面顶部会显示黄色警告横幅：

```text
⚠️ 检测到环境变量冲突
发现 X 个环境变量可能与 YKW Bridge 配置冲突
[展开] [关闭]
```

## 查看冲突详情

点击「展开」按钮后，可以看到：

| 字段 | 说明 |
|------|------|
| 变量名 | 环境变量名称 |
| 变量值 | 当前设置的值 |
| 来源 | 变量的来源位置 |

### 来源类型

| 来源 | 说明 |
|------|------|
| 用户注册表 | Windows 用户级环境变量 |
| 系统注册表 | Windows 系统级环境变量 |
| Shell 配置 | macOS/Linux 的 shell 配置文件 |
| 系统环境 | 当前进程环境变量 |

在 macOS / Linux 上，YKW Bridge 会检查当前进程环境以及常见 shell 配置文件，例如：

- `~/.bashrc`
- `~/.bash_profile`
- `~/.zshrc`
- `~/.zprofile`
- `~/.profile`
- `/etc/profile`
- `/etc/bashrc`

## 处理冲突

### 选择要删除的变量

1. 勾选要删除的环境变量
2. 或点击「全选」选择所有冲突变量

### 删除变量

1. 点击「删除选中」按钮
2. 确认删除操作
3. YKW Bridge 会先备份，再删除选中的变量

### 自动备份

删除前会自动创建 JSON 备份：

- 备份位置：`~/.ykw-bridge/backups/`
- 文件名示例：`env-backup-20260418_120000.json`
- 备份内容：变量名、变量值、来源位置

## 忽略警告

如果你确认这些环境变量不会影响当前配置，可以：

1. 点击警告横幅右侧的「关闭」
2. 本次会话暂时隐藏警告
3. 下次启动时再次检测

## 手动处理

如果不想通过 YKW Bridge 删除，也可以自己手动处理。

### Windows

1. 打开「系统属性 → 高级 → 环境变量」
2. 在用户变量或系统变量中找到目标变量
3. 删除或修改它

### macOS / Linux

1. 编辑 shell 配置文件，例如 `~/.zshrc` 或 `~/.bashrc`
2. 删除或注释对应的 `export` 语句
3. 重新加载配置，例如 `source ~/.zshrc`

## 为什么会冲突

环境变量的优先级通常高于配置文件，可能导致：

- YKW Bridge 刚写入的供应商配置被覆盖
- 请求被发到错误的端点
- 使用了错误的认证信息

## 最佳实践

1. 优先用 YKW Bridge 管理 Claude 相关配置
2. 保持 shell 配置简洁，避免重复导出同一组 `ANTHROPIC_*` 变量
3. 删除前先确认备份已经生成

## 恢复已删除的变量

如果误删了环境变量：

1. 打开 `~/.ykw-bridge/backups/`
2. 找到对应的 `env-backup-*.json`
3. 按备份内容手动恢复到系统环境
