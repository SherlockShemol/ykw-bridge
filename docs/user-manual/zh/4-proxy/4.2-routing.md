# 4.2 应用接管

## 概览

“应用接管”指的是：让某个客户端的请求先进入 YKW Bridge 本地路由服务，再由它决定最终使用哪个 Provider。

当前仅支持：

- `claude`
- `claude_desktop`

旧版其他工作流的接管逻辑已经不存在。

## 开启前提

使用应用接管前，需要先启动路由服务。

## 开启入口

在“设置 → 路由服务”中，可以分别控制：

- Claude Code 接管
- Claude Desktop 接管

## 接管后的行为

### Claude Code

YKW Bridge 会把 Claude Code 的请求改写到本地代理地址。

效果是：

- 请求可被记录
- 可参与故障转移
- 切换 Provider 时可热生效

### Claude Desktop

YKW Bridge 会写入受管 gateway 配置，并通过本地 HTTPS / gateway secret 完成接管。

效果同样包括：

- 请求日志
- 故障转移
- 热切换
- Doctor / 证书 / 启动链路联动

## 接管状态下的切换

当路由服务运行且应用已被接管时：

1. 在主界面切换 Provider
2. 代理层立即开始把新请求转发给新的 Provider
3. **不需要重启客户端**

这就是“热切换”的核心价值。

## 关闭接管

关闭接管后，YKW Bridge 会尝试恢复接管前的 live 配置或备份内容。

如果你发现没有恢复，通常是因为：

- 中途异常退出
- live 文件被其他程序改写

这时最稳妥的做法是：

1. 打开当前 Provider
2. 重新保存一次
3. 或重新开启再关闭接管

## 注意事项

- 接管依赖本地代理服务，服务未启动时开关不会产生实际效果
- Claude Desktop 的接管链路比 Claude Code 更复杂，首次使用前建议先跑一遍 Doctor
- 使用代理接管访问官方 API 时要自行评估风险，尤其是第三方反代或托管 OAuth 路径
