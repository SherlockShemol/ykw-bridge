# 4.3 故障转移

## 概览

故障转移会在当前 Provider 请求失败时，自动切换到队列中的下一个 Provider。

当前故障转移按应用分别配置，支持：

- Claude Code
- Claude Desktop

## 使用前提

要让故障转移真正生效，需要同时满足：

1. 路由服务已启动
2. 对应应用已开启接管
3. 已配置故障转移队列
4. 已启用自动故障转移

## 配置队列

在故障转移面板中：

1. 选择要配置的应用
2. 把可用 Provider 加入队列
3. 通过顺序决定优先级

排序规则：

- 越靠前优先级越高
- 当前请求失败后会按顺序尝试后面的 Provider

## 快捷操作

如果你开启了主页面的故障转移开关，Provider 卡片上也可以直接：

- 加入队列
- 移出队列

## 自动故障转移

自动故障转移关闭时：

- 只记录失败
- 不自动切换

自动故障转移开启时：

- 当前 Provider 失败后自动切到队列中的下一个

## 熔断器

YKW Bridge 会对连续失败的 Provider 进行熔断保护，避免把流量不断打到明显不可用的节点。

关键状态：

- **Closed**：正常可用
- **Open**：已熔断，暂时跳过
- **Half-Open**：恢复探测中

常见可调参数包括：

- 最大重试次数
- 首字超时
- 流式空闲超时
- 连续失败阈值
- 恢复等待时间
- 错误率阈值

Claude Code 与 Claude Desktop 当前使用同一套 Claude-like 默认参数。

## 健康状态

队列和卡片上会显示健康状态：

- 绿色：健康
- 黄色：退化/已有失败
- 红色：已熔断

## 建议

- 第一个放最稳定的主力 Provider
- 第二个放延迟和成功率都还不错的备选
- 最后再放慢但稳定的兜底 Provider

## 常见问题

### 为什么没有自动切换？

请依次检查：

- 路由服务是否运行
- 应用是否已接管
- 自动故障转移是否开启
- 队列里是否真的有备选 Provider

### 为什么切换过于频繁？

通常是：

- 主 Provider 本身就不稳定
- 熔断阈值设得太敏感

可以适当放宽阈值，或更换主 Provider。
