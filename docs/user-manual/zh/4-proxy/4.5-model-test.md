# 4.5 模型测试（Stream Check）

## 概览

模型测试会向当前 Provider 发送一条真实测试请求，用来验证：

- 模型是否存在
- 凭证是否有效
- 端点是否可达
- 延迟是否正常
- 流式首字时间是否可接受

当前专版的 Stream Check 只服务于 **Claude-like Provider**，也就是 Claude Code 与 Claude Desktop 工作流共用的一套测试配置。

## 配置项

在“设置 → 高级 → 模型测试配置”中可以调整：

| 配置项 | 说明 | 默认值 |
| --- | --- | --- |
| Claude 测试模型 | 用于测试的模型名 | 系统默认 |
| 超时时间 | 单次测试请求超时 | 45 秒 |
| 最大重试次数 | 失败后的重试次数 | 2 |
| 退化阈值 | 超过该耗时则标记为退化 | 6000ms |
| 测试提示词 | 发送给模型的短提示 | `Who are you?` |

## 建议

- 测试模型优先选便宜、快、稳定的模型
- 对 Claude 工作流，通常推荐 Haiku 一类的小模型
- 不建议把测试提示词写得太长

## 手动测试

在 Provider 卡片上点击“测试”后，YKW Bridge 会：

1. 构造一条最小化请求
2. 发到当前 Provider
3. 统计响应时间与首字时间
4. 给出健康状态

## 结果状态

常见结果：

- **健康**：请求成功且延迟在阈值内
- **退化**：请求成功，但延迟超过阈值
- **不可用**：请求失败、超时或模型不可用

## 与故障转移的关系

路由服务开启后，Stream Check 的结果会参与故障转移与健康状态判断：

- 健康节点优先保留
- 持续失败节点可能进入熔断
- 恢复中的节点会重新进行探测

## 注意事项

- 模型测试会消耗少量配额
- 如果测试模型本身填错，会出现“测试失败但实际业务可用”的情况
- Claude Code 与 Claude Desktop 当前共享同一套测试模型配置
