# 4.4 用量统计

## 概览

用量统计会聚合请求日志、Token 消耗和模型成本，帮助你观察：

- 请求量
- 成功率
- Token 用量
- 估算成本
- Provider / 模型分布

当前版本的主要数据来源有两类：

| 数据来源 | 覆盖范围 | 是否依赖代理 |
| --- | --- | --- |
| 路由请求日志 | 所有经过 YKW Bridge 代理的请求 | 是 |
| Claude 会话日志 | Claude 本地会话中的用量信息 | 否 |

## 应用筛选

当前用量页按以下维度筛选：

- 全部
- Claude
- Claude Desktop

> 这里的“Claude Desktop”主要用于区分通过 Claude Desktop 接管链路产生的代理日志。会话日志同步目前只来自 Claude 会话目录。

## 进入页面

打开“设置 → 用量”。

## 页面内容

### 汇总卡片

顶部会显示：

- 总请求数
- 总 Token
- 估算成本
- 成功率

### 趋势图

支持按时间范围查看：

- 24 小时
- 7 天
- 30 天

### 明细表

底部有三组统计：

- 请求日志
- Provider 统计
- 模型统计

## 请求日志

每条日志会展示：

- 时间
- 应用类型
- Provider
- 模型
- 输入 / 输出 Token
- 缓存命中 / 缓存创建
- 估算成本
- 延迟 / 首字时间
- HTTP 状态码

## 模型定价

你可以在高级设置中为模型维护自定义定价。

常见字段：

- 模型 ID
- 显示名称
- 输入价格
- 输出价格
- 缓存读取价格
- 缓存创建价格

YKW Bridge 会先对请求里的模型名做归一化，再匹配你的定价表。

## 关于模型名里的 `codex` / `gemini`

如果你在模型统计或定价配置里看到带有 `gpt-*-codex`、`gemini-*` 的模型名，这表示：

- 它们是 **Provider 侧真实返回的模型名称**
- 不代表 YKW Bridge 仍在支持旧版独立应用入口

当前专版仍然允许 Claude 工作流通过兼容 Provider 使用这些模型名。

## 会话日志同步

点击“同步会话日志”后，YKW Bridge 会扫描 Claude 本地会话并导入可解析的用量记录。

这个过程不需要代理开启，但需要本地确实存在 Claude 会话文件。

## 常见问题

### 为什么看不到数据？

优先检查：

- 代理是否正在记录请求
- 是否已经产生过实际请求
- Claude 会话目录里是否有可解析的日志

### 为什么成本和账单不完全一致？

常见原因：

- 某些模型还没有配置精确定价
- 你的 Provider 使用了自定义倍率
- 官方账单与本地估算的四舍五入规则不同
