# 4.1 路由服务

## 概览

路由服务会在本机启动一个 HTTP/HTTPS 代理，由它统一转发请求、记录日志、执行故障转移，并在代理接管模式下实现热切换。

当前专版的服务对象是：

- **Claude Code**
- **Claude Desktop**

## 启动方式

你可以通过两种入口启动：

1. 主界面顶部的路由开关
2. 设置页中的“路由服务”面板

## 基础配置

常见配置项：

| 配置项 | 说明 | 默认值 |
| --- | --- | --- |
| 监听地址 | 代理绑定的地址 | `127.0.0.1` |
| 监听端口 | 代理端口 | `15721` |
| 启用日志 | 是否记录请求日志 | 开启 |

修改监听地址或端口前，建议先停止路由服务，再保存并重新启动。

## 运行中信息

服务启动后，面板会显示：

- 当前监听地址
- 活跃连接数
- 累计请求数
- 成功率
- 运行时长
- 当前用于转发的 Provider
- 各应用的故障转移队列

当前 Provider 会按应用区分显示，例如：

- Claude Code
- Claude Desktop

## 请求流转

典型流程如下：

1. Claude Code 或 Claude Desktop 发起请求
2. YKW Bridge 记录请求日志与用量
3. 根据当前 Provider 和故障转移策略转发
4. 返回响应给客户端

## 代理接管

启动路由服务后，若为某个应用开启“接管”：

### Claude Code

YKW Bridge 会把 Claude 的请求地址改写到本地代理地址。

### Claude Desktop

YKW Bridge 会写入受管的 Claude Desktop gateway 配置，并通过本地证书与本地网关完成转发。

> Claude Desktop 的接管链路包含证书、gateway secret、launch shim、watchdog 等辅助能力，所以它不是简单地改一条 Base URL。

## API 格式转换

Claude 工作流里如果某个 Provider 使用的不是原生 Anthropic Messages 格式，代理层会自动做格式转换。

当前支持的主要格式：

- Anthropic Messages：直通
- OpenAI Chat Completions：请求/响应双向转换
- OpenAI Responses：请求/响应双向转换

这也是 Claude 工作流能使用更多第三方 Provider 的关键。

## 停止服务

停止路由服务后，YKW Bridge 会尽量：

1. 恢复 live 配置
2. 保留备份
3. 关闭当前监听

如果某个应用启用了代理接管，建议正常关闭服务，而不是直接强杀进程。

## 日志与查看位置

请求日志会出现在“设置 → 用量”中，可用于：

- 查看请求明细
- 估算成本
- 排查失败请求
- 观察故障转移情况

## 常见问题

### 端口被占用

如果提示地址已被占用：

1. 改用其他端口
2. 或结束占用端口的程序

### 服务启动失败

优先检查：

- 监听地址和端口是否合法
- 证书/权限是否正常（尤其是 Claude Desktop）
- 是否已有旧实例未退出
