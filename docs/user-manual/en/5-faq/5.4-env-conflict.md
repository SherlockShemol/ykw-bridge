# 5.4 Environment Variable Conflicts

## Overview

YKW Bridge automatically checks for conflicts between system environment variables and the current Claude workflow config so settings are not overwritten unexpectedly.

This conflict check now targets only `Claude` / `Claude Desktop`. The manual no longer documents old multi-app environment variable flows.

**Currently scanned variable scope**:

- `ANTHROPIC_API_KEY`
- `ANTHROPIC_AUTH_TOKEN`
- `ANTHROPIC_BASE_URL`
- Other variables whose names contain `ANTHROPIC`

## Viewing Conflict Details

After clicking **Expand**, you can inspect:

| Field | Description |
|------|------|
| Variable Name | Environment variable name |
| Variable Value | Current value |
| Source | Where the variable came from |

### Source Types

| Source | Description |
|------|------|
| User Registry | Windows user-level environment variables |
| System Registry | Windows system-level environment variables |
| Shell Config | macOS/Linux shell config files |
| Process Environment | Current process environment variables |

On macOS / Linux, YKW Bridge checks the current process environment plus common shell config files such as:

- `~/.bashrc`
- `~/.bash_profile`
- `~/.zshrc`
- `~/.zprofile`
- `~/.profile`
- `/etc/profile`
- `/etc/bashrc`

## Handling Conflicts

### Choose Variables to Remove

1. Select the environment variables you want to delete
2. Or use **Select All** to choose every conflict

### Delete Variables

1. Click **Delete Selected**
2. Confirm the action
3. YKW Bridge creates a backup first, then deletes the selected variables

### Automatic Backup

Before deletion, YKW Bridge creates a JSON backup:

- Backup location: `~/.ykw-bridge/backups/`
- Example filename: `env-backup-20260418_120000.json`
- Backup content: variable names, values, and source locations

## Ignore the Warning

If you are sure those variables do not affect your current config:

1. Click **Close** on the right side of the banner
2. The warning stays hidden for the current session
3. Detection runs again on next startup

## Manual Handling

If you do not want YKW Bridge to delete the variables, you can handle them manually.

### Windows

1. Open **System Properties â†’ Advanced â†’ Environment Variables**
2. Find the target variable in user or system variables
3. Delete or modify it

### macOS / Linux

1. Edit a shell config file such as `~/.zshrc` or `~/.bashrc`
2. Delete or comment out the relevant `export` line
3. Reload the shell config, for example `source ~/.zshrc`

## Why Conflicts Matter

Environment variables usually have higher priority than config files, which can cause:

- Provider config written by YKW Bridge to be overwritten
- Requests to go to the wrong endpoint
- Incorrect authentication to be used

## Best Practices

1. Prefer managing Claude-related config inside YKW Bridge
2. Keep shell config clean and avoid exporting overlapping `ANTHROPIC_*` variables repeatedly
3. Confirm the backup has been created before deleting anything

## Restoring Deleted Variables

If you removed an environment variable by mistake:

1. Open `~/.ykw-bridge/backups/`
2. Find the relevant `env-backup-*.json`
3. Restore the variables manually based on the backup content
