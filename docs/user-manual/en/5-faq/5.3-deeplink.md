# 5.3 Deep Link Protocol

## Overview

YKW Bridge now uses the `ykwbridge://` deep link protocol for one-click import of providers, prompts, MCP config, and skill repositories.

**Common scenarios**:

- Sharing provider or MCP config with a team
- Offering one-click import in tutorials
- Reusing the same setup across devices

## Protocol Format

### V1 Protocol

```text
ykwbridge://v1/import?resource={type}&...
```

### Common Parameters

| Parameter | Required | Description |
|------|------|------|
| `resource` | Yes | Resource type: `provider` / `prompt` / `mcp` / `skill` |

## Parameters by Resource Type

### Provider (`resource=provider`)

| Parameter | Required | Description |
|------|------|------|
| `app` | Yes | `claude` / `claude_desktop` |
| `name` | Yes | Provider name |
| `endpoint` | No | API endpoint, supports comma-separated multiple URLs |
| `apiKey` | No | API key |
| `homepage` | No | Homepage URL |
| `model` | No | Default model |
| `haikuModel` | No | Default Claude Haiku model |
| `sonnetModel` | No | Default Claude Sonnet model |
| `opusModel` | No | Default Claude Opus model |
| `notes` | No | Notes |
| `icon` | No | Icon name |
| `config` | No | Base64-encoded inline config |
| `configFormat` | No | `json` / `toml` |
| `enabled` | No | Whether to enable immediately after import |
| `usageScript` | No | Base64-encoded usage query script |
| `usageEnabled` | No | Whether usage query is enabled |
| `usageApiKey` | No | Usage-query API key |
| `usageBaseUrl` | No | Usage-query base URL |
| `usageAccessToken` | No | Usage-query access token |
| `usageUserId` | No | Usage-query user ID |
| `usageAutoInterval` | No | Auto-query interval in minutes |

Notes:

- `endpoint`, `apiKey`, and `homepage` can be passed directly in the URL
- You can also pass a Base64 inline config via `config` and let YKW Bridge backfill values from it
- This manual no longer recommends remote `configUrl` as the primary path

### Prompt (`resource=prompt`)

| Parameter | Required | Description |
|------|------|------|
| `app` | Yes | `claude` / `claude_desktop` |
| `name` | Yes | Prompt name |
| `content` | Yes | Base64-encoded prompt content |
| `description` | No | Description |
| `enabled` | No | Whether to enable immediately after import |

### MCP (`resource=mcp`)

| Parameter | Required | Description |
|------|------|------|
| `apps` | Yes | Currently only `claude` is supported |
| `config` | Yes | Base64-encoded JSON containing `mcpServers` |
| `enabled` | No | Whether to enable |

### Skill Repository (`resource=skill`)

| Parameter | Required | Description |
|------|------|------|
| `repo` | Yes | Repository in `owner/name` format |
| `directory` | No | Directory path inside the repo |
| `branch` | No | Branch name |

Skill deep links do not need a separate `app` parameter. They currently import into Claude Skills repository management.

## Import Examples

### Import a Claude Provider

```text
ykwbridge://v1/import?resource=provider&app=claude&name=My%20Provider&endpoint=https%3A%2F%2Fapi.example.com&apiKey=sk-xxx
```

### Import a Claude Desktop Provider

```text
ykwbridge://v1/import?resource=provider&app=claude_desktop&name=Desktop%20Provider&endpoint=https%3A%2F%2Fapi.example.com&apiKey=sk-xxx
```

### Import MCP

```text
ykwbridge://v1/import?resource=mcp&apps=claude&config=BASE64_ENCODED_JSON
```

### Import a Prompt

```text
ykwbridge://v1/import?resource=prompt&app=claude&name=Code%20Review&content=BASE64_ENCODED_CONTENT
```

### Import a Skill Repository

```text
ykwbridge://v1/import?resource=skill&repo=owner/repo&directory=skills/my-skill&branch=main
```

## How to Generate Links

### Manual Generation

1. Prepare the parameters
2. Build a `ykwbridge://v1/import?...` URL
3. URL-encode special characters
4. Base64-encode fields such as `content`, `config`, or `usageScript`

### JavaScript Example

```javascript
const params = new URLSearchParams({
  resource: "provider",
  app: "claude",
  name: "My Provider",
  endpoint: "https://api.example.com",
  apiKey: "sk-xxx",
});

const url = `ykwbridge://v1/import?${params.toString()}`;
```

### Base64 Example

```javascript
const promptContent = Buffer.from("# Role\nYou are a reviewer", "utf8").toString("base64");
```

## Using Deep Links

When you click a deep link, YKW Bridge:

1. Validates the protocol and parameters
2. Opens the import confirmation dialog
3. Shows a preview of the config
4. Waits for your confirmation before importing

**Security tip**: only import links from trusted sources.

## Protocol Registration

### Automatic Registration

YKW Bridge automatically registers the `ykwbridge://` protocol during installation.

### Manual Check

If the protocol does not work:

- macOS: reinstall the app or rerun the app registration flow
- Windows: check `HKEY_CLASSES_ROOT\ykwbridge`
- Linux: check the protocol declaration in the `.desktop` file

## Security Suggestions

Deep links may contain sensitive information such as:

- API keys
- Usage query scripts
- Third-party endpoint addresses

Suggestions:

1. Do not publicly share links containing real credentials
2. Replace sensitive values before sharing
3. Review the preview dialog before importing

## Troubleshooting

### The Link Does Not Open

Check first:

1. Whether YKW Bridge is installed
2. Whether `ykwbridge://` is registered
3. Whether the link format is correct

### Import Fails

**Common causes**:

- Invalid Base64 encoding
- Invalid JSON / TOML content
- Missing required fields
- `app` or `apps` still uses unsupported old values

**Fix**:

1. Recheck the original content
2. Re-encode the Base64 fields
3. Make sure `provider` / `prompt` uses only `claude` or `claude_desktop`
4. Make sure `mcp.apps` uses only `claude`
