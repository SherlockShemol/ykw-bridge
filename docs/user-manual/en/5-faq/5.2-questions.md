# 5.2 Frequently Asked Questions

## Installation

### macOS Installation

The macOS build of YKW Bridge is signed and notarized. Under normal conditions it should install directly. If something goes wrong, start by re-downloading the latest installer from the [Releases page](https://github.com/SherlockShemol/ykw-bridge/releases).

### App Does Not Start on Windows

**Common causes**:

- Missing WebView2 runtime
- Antivirus blocking the app

**Fix**:

1. Install [Microsoft Edge WebView2](https://developer.microsoft.com/en-us/microsoft-edge/webview2/)
2. Add YKW Bridge to the antivirus allowlist

### Linux Startup Error

**Problem**: AppImage does not launch

**Fix**:

```bash
chmod +x YKW-Bridge-*.AppImage
./YKW-Bridge-*.AppImage --no-sandbox
```

## Provider Issues

### Switching Providers Does Not Take Effect

**Reason**: Different workflows read config at different times.

**Fix**:

- Claude Code normal mode: close and reopen the terminal, or restart the Claude Code process inside the IDE
- Claude Code takeover / proxy mode: in most cases the change takes effect hot
- Claude Desktop managed profile: if the current app instance still holds old config, quit Claude.app and open it again

### API Key Is Invalid

**Check**:

1. Make sure the key was copied without extra spaces or line breaks
2. Make sure the key is not expired or revoked
3. Make sure the endpoint matches the auth mode
4. Use speed test or stream check to verify real connectivity

### How Do I Restore Official Login?

**Steps**:

1. Select Claude's **Official Login** preset
2. Click **Enable**
3. Reopen the corresponding Claude workflow
4. Complete the official login flow

> `codex_oauth` is still kept, but it is a provider path that can be attached to a Claude workflow, not a standalone Codex app.

## Proxy Issues

### Routing Service Fails to Start

**Common cause**: the port is already in use

**Fix**:

1. Check port usage:

   ```bash
   # macOS/Linux
   lsof -i :49152

   # Windows
   netstat -ano | findstr :49152
   ```

2. Stop the process occupying that port
3. Or restore the default port in **Settings → Routing Service**

### Requests Time Out in Proxy Mode

**Common causes**:

- Unstable network
- Upstream provider outage
- Incorrect proxy config

**Fix**:

1. Confirm local network connectivity first
2. Temporarily disable the proxy and compare by hitting the provider directly
3. Check endpoint, auth, and model config on the provider card

### Config Was Not Restored After Stopping the Proxy

**Common cause**: the proxy exited abnormally before restore finished.

**Fix**:

1. Open the current provider
2. Check endpoint and auth fields
3. Save it again so live config returns to the current database state

## Failover Issues

### Why Did Failover Not Trigger?

**Checklist**:

- [ ] The routing service is running
- [ ] App takeover is enabled
- [ ] Automatic failover is enabled
- [ ] The queue really contains fallback providers

### Why Does It Switch Too Often?

**Common causes**:

- The primary provider is unstable
- The breaker threshold is too aggressive

**Fix**:

1. Check the primary provider first
2. Relax the failure thresholds a little
3. Replace the primary provider if needed

### What If All Providers Are Tripped?

**Fix**:

1. Wait for the breaker recovery window to expire
2. Or restart the routing service to reset the state

## Data Issues

### Configuration Looks Missing

**Common causes**:

- The data directory was deleted
- The database is corrupted
- You are still looking at the old `.cc-switch` location by mistake

**Fix**:

1. Check whether `~/.ykw-bridge/` exists
2. Check `~/.ykw-bridge/backups/` for recoverable backups
3. Re-import from an exported config file if necessary

### Import Fails

**Common causes**:

- Invalid file format
- Incomplete content
- Incompatible data version

**Fix**:

1. Confirm the file was exported by YKW Bridge
2. Open the JSON in a text editor and make sure it is complete
3. Re-export or try a backup closer to the current version

### Usage Statistics Are Empty

**Checklist**:

- [ ] The routing service is running
- [ ] App takeover is enabled
- [ ] Logging is enabled
- [ ] Real requests actually passed through the current route

## Quota and Balance

### Why Do Some Providers Show Quota Automatically While Others Need Manual Setup?

The current manuals keep automatic quota querying for:

- Claude official login
- GitHub Copilot
- Codex OAuth reverse proxy

Other providers, including Token Plan, third-party balance queries, and relay services, need to be enabled manually in the provider card's **Usage Query** panel. The reason is simple: the same request URL can expose both plan quota and account balance modes, and YKW Bridge will not guess which one you want.

### Official Subscription Provider Shows No Quota

**Check**:

1. Make sure the provider is the currently active one
2. For Copilot / Codex OAuth, confirm the OAuth token is still valid
3. If the card says **Session Expired**, log in again in the **OAuth Auth Center**
4. Click the refresh button on the card and query again

### Manually Enabled Usage Still Does Not Show

**Check**:

1. Make sure **Enable Usage Query** is turned on
2. Make sure the correct template was selected and saved
3. Use **Test Script** to inspect the real error
4. Make sure the provider is currently active

## Codex OAuth Reverse Proxy

### What Are the Risks of Enabling Codex OAuth Reverse Proxy?

Codex OAuth reverse proxy uses a reverse-engineered OAuth flow to access Codex service through your ChatGPT account. This may violate OpenAI's Terms of Service and may carry risk of account limits, behavior changes, or long-term instability. Evaluate the risk yourself before enabling it.

See the full disclaimer in [2.1 Add Provider → Codex OAuth Reverse Proxy](../2-providers/2.1-add.md#codex-oauth-reverse-proxy-claude-provider).

### How Do I Log In to Codex OAuth?

For the full Device Code login flow, the two entry points (Add Provider panel / OAuth Auth Center), multi-account management, and common failure cases, see [2.1 Add Provider → Codex OAuth Reverse Proxy (Claude Provider)](../2-providers/2.1-add.md#codex-oauth-reverse-proxy-claude-provider).

### Codex OAuth Logged In but No Quota Shown

**Fix**:

1. Confirm you completed login in the **OAuth Auth Center**
2. Check whether the token is still valid
3. If the card says **Session Expired**, remove that account and log in again

## Other Issues

### Tray Icon Does Not Show

**macOS**:

- Check the menu bar icon visibility settings

**Windows**:

- Check whether the taskbar icon is collapsed into the hidden area

**Linux**:

- Make sure the system tray support package is installed, such as `libappindicator`

### UI Looks Wrong

**Fix**:

1. Retry after switching theme
2. Restart the app
3. Delete `~/.ykw-bridge/settings.json` to reset device-level settings

### Update Failed

**Fix**:

1. Check network connectivity
2. Download and install the latest release again from the Releases page

## Lightweight Mode

### How Do I Enter Lightweight Mode?

Toggle **Lightweight Mode** from the system tray menu. The main window closes, but YKW Bridge keeps running in the tray. Toggle it again or click **Open Main Window** to return.

### Does Lightweight Mode Use Less Memory?

Yes. Lightweight Mode destroys the main window and WebView, which usually reduces memory use noticeably while keeping tray functionality.

### Can a Deep Link Still Reopen the Main Window in Lightweight Mode?

Yes. Clicking a `ykwbridge://` link makes YKW Bridge rebuild the main window on demand and open the import confirmation dialog. The first wake-up can feel slightly slower, then it returns to normal.

## Getting Help

### Opening an Issue

If the methods above still do not solve the problem:

1. Visit [GitHub Issues](https://github.com/SherlockShemol/ykw-bridge/issues)
2. Search for an existing similar issue
3. Open a new one if needed
4. Include:
   - OS and version
   - YKW Bridge version
   - Reproduction steps
   - Relevant errors

### What Diagnostics Help Most

It is helpful to also include relevant diagnostics such as:

- Crash logs or backup files under `~/.ykw-bridge/`
- Screenshots of provider card errors
- Proxy or stream-check error output
