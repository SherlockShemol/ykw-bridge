# 5.1 Configuration Files

## YKW Bridge Data Directory

### Default Location

The current default data directory is: `~/.ykw-bridge/`

Unless you override specific paths in settings, the database, device-level settings, Skills, Workspace, backups, and the Claude Desktop managed profile all live here.

### Common Directory Structure

```text
~/.ykw-bridge/
├── ykw-bridge.db              # Main SQLite database
├── settings.json              # Device-level settings
├── config.json                # Aggregated config file
├── backups/                   # Backups for DB/import/env-var deletion
├── skills/                    # Skills single source of truth (SSOT)
├── skill-backups/             # Skill uninstall backups
├── workspace/                 # Workspace files
│   └── memory/                # Daily Memory
├── claude-desktop/
│   └── profile/               # Claude Desktop managed profile (default)
└── crash.log                  # Crash log (created on demand)
```

### What the Database Stores

`ykw-bridge.db` is the current main database. It primarily stores:

- Provider config and custom endpoints
- MCP servers
- Prompt presets
- Skill install state and repository info
- Proxy, failover, and request logs
- Model pricing and health state

### `settings.json`

`settings.json` stores device-level settings and does not sync across devices together with the database. Typical fields look like:

```json
{
  "visibleApps": {
    "claude": true,
    "claudeDesktop": true
  },
  "claudeConfigDir": null,
  "claudeDesktopAppPath": null,
  "claudeDesktopProfileDir": null,
  "launchOnStartup": false
}
```

### `backups/`

The `backups/` directory can contain several kinds of backups:

- Automatic backups created before imports
- Safety backups created before database restore
- `env-backup-*.json` files created before deleting conflicting environment variables

## Claude Code Configuration

### Default Directory

Default path: `~/.claude/`

### Common Files

```text
~/.claude/
├── settings.json     # Claude Code main config
├── CLAUDE.md         # Prompt / system instructions
└── skills/           # Synced skills
```

### MCP Config File

By default, Claude Code MCP config is written to: `~/.claude.json`

If you override the Claude Code config directory in settings, the MCP file follows that override and is written as a sibling `*.json` sidecar file instead of staying fixed at the default path.

## Claude Desktop Managed Profile

### Default Directory

Default path: `~/.ykw-bridge/claude-desktop/profile/`

You can also override this path in settings.

### What Usually Lives There

The managed profile keeps files needed by the Claude Desktop workflow, such as:

- Managed config
- `CLAUDE.md`
- `skills/`
- Local gateway / certificate related files
- Claude Desktop local session directories

This is the managed directory for the Claude Desktop workflow, not a standalone third-party app config directory.

## Workspace and Daily Memory

Default paths:

- `~/.ykw-bridge/workspace/`
- `~/.ykw-bridge/workspace/memory/`

Workspace panel files and Daily Memory are stored here.

## Configuration Priority

You can think of YKW Bridge's current behavior as three layers:

1. **Database**: the main data source
2. **Claude / Claude Desktop live files**: written when providers, prompts, MCP, or skills change
3. **Backfill reads**: when editing current config, YKW Bridge tries to read existing live files back into the UI

## Manual Editing Guidance

### Safe to Edit Manually

- Claude Code config files
- Plain-text files in the Claude Desktop managed profile when you understand their purpose
- YKW Bridge `settings.json`

### Not Recommended to Edit Manually

- `ykw-bridge.db`
- Files inside `backups/`
- Claude Desktop managed gateway files whose purpose you do not fully understand

### How to Resync After Manual Edits

If you manually edit Claude-related live config:

1. Open YKW Bridge
2. Go to the related provider or config panel
3. Verify the backfilled content looks correct
4. Save once more so the database and live files are aligned again

## Configuration Migration

### Legacy `.cc-switch`

Older versions may have used `~/.cc-switch/`. The current default directory is now `~/.ykw-bridge/`.

If the new directory does not exist but the old one does, YKW Bridge may automatically import legacy data for compatibility. In practice:

- `~/.cc-switch/` is a legacy compatibility source
- `~/.ykw-bridge/` is the current default location

### Cross-Device Migration

You can migrate in three ways:

1. Export and import configuration
2. Use WebDAV sync
3. Manually back up and restore the parts you need from `~/.ykw-bridge/`

## Backup Recommendations

### Export Regularly

It is a good idea to export config regularly from the data-management panel in settings, especially:

- Before large provider changes
- Before bulk Prompt / MCP / Skills changes
- Before switching devices

### What Export Usually Contains

- Provider config
- MCP servers
- Prompt presets
- Skill metadata
- App data suitable for sync

### What You Should Not Rely on Alone

Do not rely solely on runtime logs, temporary state files, or device-specific path overrides as your only recovery strategy across devices.
