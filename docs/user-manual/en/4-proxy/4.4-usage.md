# 4.4 Usage Statistics

## Overview

Usage statistics aggregate request logs, token consumption, and model cost so you can observe:

- Request volume
- Success rate
- Token usage
- Estimated cost
- Provider and model distribution

The current version mainly uses two data sources:

| Data Source | Coverage | Requires Proxy |
| --- | --- | --- |
| Routed request logs | All requests that pass through YKW Bridge | Yes |
| Claude session logs | Usage information from local Claude sessions | No |

## App Filter

The usage page currently filters by:

- All
- Claude
- Claude Desktop

> "Claude Desktop" here mainly distinguishes proxy logs produced through the Claude Desktop takeover chain. Session-log sync currently imports only from Claude session directories.

## Open the Page

Open **Settings → Usage**.

## Page Content

### Summary Cards

At the top you will see:

- Total requests
- Total tokens
- Estimated cost
- Success rate

### Trend Chart

You can inspect trends over:

- 24 hours
- 7 days
- 30 days

### Detail Tables

At the bottom there are three groups of statistics:

- Request logs
- Provider statistics
- Model statistics

## Request Logs

Each log entry shows:

- Time
- App type
- Provider
- Model
- Input and output tokens
- Cache read / cache creation
- Estimated cost
- Latency / time to first byte
- HTTP status

## Model Pricing

You can maintain custom pricing for models in advanced settings.

Common fields:

- Model ID
- Display name
- Input price
- Output price
- Cache read price
- Cache creation price

YKW Bridge normalizes the model name from the request first, then matches it against your pricing table.

## About Model Names That Include `codex` or `gemini`

If you see model names such as `gpt-*-codex` or `gemini-*` in model statistics or pricing configuration, that means:

- They are **real model names returned by the provider**
- They do **not** mean YKW Bridge still exposes the old standalone app entry points

The current edition still allows Claude workflows to use those model names through compatible providers.

## Session Log Sync

When you click **Sync Session Logs**, YKW Bridge scans local Claude sessions and imports any parsable usage records.

This does not require the proxy to be running, but it does require local Claude session files to exist.

## Common Issues

### Why Is There No Data?

Check first:

- Whether the proxy is currently recording requests
- Whether any real requests have passed through it
- Whether the Claude session directory contains parsable logs

### Why Does Cost Not Exactly Match the Official Bill?

Common reasons:

- Some models still do not have exact pricing configured
- Your provider uses custom multipliers
- Official billing and local estimation may round values differently
