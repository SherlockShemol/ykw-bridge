# 4.3 Failover

## Overview

Failover automatically switches to the next provider in the queue when the current provider request fails.

Failover is configured per app and currently supports:

- Claude Code
- Claude Desktop

## Prerequisites

For failover to actually work, all of the following must be true:

1. The routing service is running
2. Takeover is enabled for the relevant app
3. A failover queue has been configured
4. Automatic failover is enabled

## Configure the Queue

In the failover panel:

1. Choose which app you want to configure
2. Add available providers into the queue
3. Use order to define priority

Ordering rules:

- Earlier entries have higher priority
- When the current request fails, later entries are tried in order

## Quick Actions

If the failover toggle is enabled on the main page, provider cards can also directly:

- Add a provider to the queue
- Remove a provider from the queue

## Automatic Failover

When automatic failover is off:

- Failures are logged
- Providers are not switched automatically

When automatic failover is on:

- A failing provider is automatically replaced by the next provider in the queue

## Circuit Breaker

YKW Bridge applies circuit-breaker protection to providers with repeated failures so traffic does not keep hitting clearly unhealthy endpoints.

Key states:

- **Closed**: healthy and available
- **Open**: temporarily tripped and skipped
- **Half-Open**: under recovery probing

Common tunables include:

- Max retries
- Time to first byte timeout
- Stream idle timeout
- Consecutive failure threshold
- Recovery wait time
- Error rate threshold

Claude Code and Claude Desktop currently share the same Claude-like default tuning.

## Health Status

Health state is shown in queues and cards:

- Green: healthy
- Yellow: degraded / some failures
- Red: tripped

## Recommendations

- Put your most stable primary provider first
- Put a reasonably fast and reliable fallback second
- Keep a slower but stable safety net at the end

## Common Issues

### Why Did It Not Switch Automatically?

Check in order:

- Is the routing service running?
- Is the app taken over?
- Is automatic failover enabled?
- Does the queue really contain fallback providers?

### Why Does It Switch Too Often?

Usually:

- The main provider is unstable
- The circuit-breaker thresholds are too sensitive

You can relax the thresholds slightly or replace the primary provider.
