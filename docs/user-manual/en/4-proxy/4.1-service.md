# 4.1 Routing Service

## Overview

The routing service starts a local HTTP/HTTPS proxy that forwards requests, records logs, performs failover, and enables hot switching when takeover mode is enabled.

The current service targets are:

- **Claude Code**
- **Claude Desktop**

## How to Start It

You can start it from two places:

1. The routing toggle at the top of the main page
2. The **Routing Service** panel in settings

## Basic Configuration

Common settings:

| Setting | Description | Default |
| --- | --- | --- |
| Listen Address | Address the proxy binds to | `127.0.0.1` |
| Listen Port | Proxy port | `15721` |
| Enable Logging | Whether request logs are recorded | Enabled |

If you want to change the listen address or port, it is best to stop the routing service first, save the change, then start it again.

## Runtime Information

After startup, the panel shows:

- Current listen address
- Active connections
- Total requests
- Success rate
- Uptime
- Current forwarding provider
- Failover queues for each app

The current provider is shown separately per app, for example:

- Claude Code
- Claude Desktop

## Request Flow

Typical flow:

1. Claude Code or Claude Desktop sends a request
2. YKW Bridge records request logs and usage data
3. The request is forwarded based on the current provider and failover strategy
4. The response is returned to the client

## Proxy Takeover

Once the routing service is running, you can enable **takeover** for a specific app.

### Claude Code

YKW Bridge rewrites Claude Code requests to the local proxy address.

### Claude Desktop

YKW Bridge writes a managed Claude Desktop gateway configuration and forwards through local certificates and the local gateway.

> The Claude Desktop takeover chain includes certificates, gateway secret handling, launch shim behavior, watchdog integration, and related helpers, so it is more than simply replacing a base URL.

## API Format Conversion

If a provider in the Claude workflow does not use native Anthropic Messages format, the proxy layer automatically converts the request and response format.

Currently supported formats:

- Anthropic Messages: pass-through
- OpenAI Chat Completions: bidirectional conversion
- OpenAI Responses: bidirectional conversion

This is a key reason why the Claude workflow can work with more third-party providers.

## Stop the Service

When stopping the routing service, YKW Bridge tries to:

1. Restore live config
2. Keep backups intact
3. Close the current listener

If takeover is enabled for an app, stopping the service cleanly is safer than force-killing the process.

## Logs and Where to View Them

Request logs appear in **Settings → Usage** and can be used to:

- Inspect request details
- Estimate cost
- Troubleshoot failed requests
- Observe failover behavior

## Common Issues

### Port Already in Use

If the address is already occupied:

1. Use another port
2. Or stop the process currently using that port

### Service Fails to Start

Check first:

- Whether the listen address and port are valid
- Whether certificate or permission state is healthy, especially for Claude Desktop
- Whether an old instance is still running
