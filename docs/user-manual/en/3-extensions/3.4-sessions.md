# 3.4 Session Manager

## Overview

The current session manager only browses and manages **Claude Code** local session logs.

Session files come from Claude's project session directory:

- JSONL session files under `~/.claude/projects/`

Other legacy workflow session entries have been removed from the current edition.

## Open the Session Manager

Click **Sessions** in the top navigation bar.

## What You Can Do

- Browse recent sessions
- Search by title, directory, summary, or session ID
- View the full message history
- Copy the resume command
- Delete single sessions or multiple local session files in bulk

## Interface Structure

### Left List

Each session shows:

- Title
- Last active time
- Project directory

### Right Detail Panel

Once selected, you can view:

- Full timestamp
- Project directory
- Resume command
- Full message history

## Search and Filter

The current filters are reduced to:

- **All**
- **Claude**

Press `Esc` to clear the search box quickly.

## Resume a Session

After clicking **Resume**:

- On macOS, YKW Bridge tries to launch a terminal directly
- On other platforms, it prefers copying the resume command

If terminal launch fails, it falls back to copying the command.

## Delete Sessions

### Delete a Single Session

Use the delete action in the detail panel. After confirmation, the corresponding local session file is removed.

### Bulk Delete

1. Enter bulk mode
2. Select multiple sessions
3. Click bulk delete

## Directory Picker

The current directory picker is primarily designed to make **Claude session resume** more reliable. Older workflow resume paths no longer exist.

## Notes

- Sessions are sorted by last active time in descending order
- Only writable local Claude session files can be deleted
- Read-only entries without local source files cannot be deleted
