# 3.1 MCP Server Management

## Overview

YKW Bridge currently manages MCP servers only for **Claude Code**.

MCP configuration is stored in the YKW Bridge database and, when enabled, synced into the `mcpServers` section of `~/.claude.json`. The current UI keeps only the Claude workflow MCP toggle.

## Open the MCP Panel

Click **MCP** in the top navigation bar.

## Add an MCP Server

### Use a Preset Template

1. Click **+** in the upper-right corner
2. Select a template from the preset dropdown
3. Adjust the command, arguments, or URL as needed
4. Click **Save**

Common presets include:

- `fetch`
- `time`
- `memory`
- `sequential-thinking`
- `context7`

### Custom Configuration

Common fields in custom mode:

| Field | Required | Description |
| --- | --- | --- |
| Server ID | Yes | Unique identifier |
| Name | No | Display name |
| Description | No | Human-readable description |
| Transport Type | Yes | `stdio` / `http` / `sse` |
| Command | Required for `stdio` | Launch command |
| Arguments | No | Command arguments |
| URL | Required for `http` / `sse` | Remote service address |
| Headers | No | HTTP headers |
| Environment Variables | No | Environment variables passed to the MCP process |

## Enable and Sync

The MCP panel now keeps only the **Claude** toggle.

When enabled:

1. The database records the server as enabled for Claude
2. The config is written to `~/.claude.json`
3. Claude Code loads it the next time it starts or reloads configuration

When disabled:

1. The database state is updated
2. The server is removed from `~/.claude.json`

## Import Existing Configuration

If you already configured MCP servers locally for Claude, click **Import**:

1. Read the current `~/.claude.json`
2. Import its `mcpServers` entries into YKW Bridge
3. Manage future changes in YKW Bridge from then on

## Configuration Examples

### `stdio`

```json
{
  "command": "uvx",
  "args": ["mcp-server-fetch"]
}
```

### `http`

```json
{
  "url": "http://localhost:8080/mcp"
}
```

### Example Claude MCP File

```json
{
  "mcpServers": {
    "mcp-fetch": {
      "command": "uvx",
      "args": ["mcp-server-fetch"]
    }
  }
}
```

## Deep Link Import

MCP deep links currently support only:

- `resource=mcp`
- `apps=claude`

Imports fail if the link still uses old multi-app identifiers.

## Notes

- In `stdio` mode, the command must already be installed and available in `PATH`
- After changing MCP config, it is best to reopen Claude Code or start a new session
- If the local Claude config path does not exist, enabling the toggle may not write the live file, but the database entry is still preserved

## Common Issues

### Server Fails to Start

Check first:

- Whether the command is installed
- Whether the arguments are correct
- Whether the required local environment variables are present

### Configuration Does Not Take Effect

Check first:

- Whether the Claude toggle is enabled
- Whether `~/.claude.json` was overwritten by another tool
- Whether Claude Code has already reloaded configuration
