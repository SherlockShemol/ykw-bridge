# 3.3 Skills Management

## Overview

Skills are reusable capability extensions. The current edition keeps Skills management only for the **Claude workflow**.

YKW Bridge manages Skills through a single source of truth (SSOT):

- Default SSOT: `~/.ykw-bridge/skills/`
- Backup directory: `~/.ykw-bridge/skill-backups/`

## Open the Skills Page

Click **Skills** in the top navigation bar.

## Discover Available Skills

YKW Bridge can discover skills from preset repositories, your own GitHub repositories, and the public `skills.sh` catalog.

Common actions:

1. Search by name or description
2. Filter by installed / not installed
3. Click **Refresh** to rescan remote repositories

## Install Skills

1. Find the target skill
2. Click **Install**
3. Wait for installation to finish

After installation, the skill is written to the SSOT and synced into the Claude workflow directory.

## Uninstall and Restore

YKW Bridge creates a backup before uninstalling a skill.

After uninstall:

1. The skill is removed from the SSOT
2. The synced copy is removed from the Claude workflow directory
3. The install record is removed from the database

To restore:

1. Open **Restore from Backup**
2. Select a backup entry
3. Restore it and enable it again

## Repository Management

You can add custom repositories with:

- Owner
- Repository
- Branch
- Subdirectory (optional)

Removing a repository only stops future updates. It does not automatically delete already installed skills.

## ZIP Install and Import

In addition to online repositories, YKW Bridge also supports:

- Installing from ZIP files
- Scanning unmanaged local skills
- Importing existing Claude-side skill directories

## Storage Location Switching

The source storage location for skills can be switched between:

| Location | Description |
| --- | --- |
| `~/.ykw-bridge/skills/` | Default location managed by YKW Bridge |
| `~/.agents/skills/` | Community-style shared directory for multi-tool setups |

Switching storage locations does not lose the current install state.

## Notes

- Only Claude-side skill sync is kept in the current edition
- Sync method (symlink / copy) and source storage location are separate settings
- If a remote repository is unreachable, already installed skills continue to work
