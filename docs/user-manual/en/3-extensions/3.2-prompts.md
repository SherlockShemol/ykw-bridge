# 3.2 Prompts Management

## Overview

Prompts manage reusable system prompt presets. YKW Bridge currently supports separate prompt sets for:

- **Claude Code**
- **Claude Desktop**

These two targets are independent from each other.

## Open the Panel

Click **Prompts** in the top navigation bar.

## Create a Preset

1. Click **+**
2. Enter a preset name
3. Write the prompt in the Markdown editor
4. Click **Save**

The editor supports:

- Markdown syntax highlighting
- Live preview
- Common formatting shortcuts

## Activate a Preset

Enable a preset using the toggle on its card.

Only one preset can be active per target at a time. Enabling a new preset automatically disables the previous one for that target.

## Sync Targets

Once activated, YKW Bridge writes the content back to the corresponding live file:

| Target | Sync File |
| --- | --- |
| Claude Code | `~/.claude/CLAUDE.md` |
| Claude Desktop | `CLAUDE.md` inside the Claude Desktop profile directory |

> The Claude Desktop profile path depends on platform and any custom override from settings. Use the app's status or Doctor output as the source of truth.

## Edit and Delete

### Edit

1. Click **Edit**
2. Update the name or content
3. Click **Save**

If you edit the currently active preset, the live file is updated immediately after saving.

### Delete

1. Click **Delete**
2. Confirm the action

You cannot delete the currently active preset directly. Disable it first or switch to another preset.

## Smart Backfill

Before switching presets, YKW Bridge reads the current live file and backfills any manual external edits into the current preset so they are not lost.

Typical trigger points:

- When switching to another preset
- When editing the currently active preset
- During first-run import of an existing live file

## Deep Link

Prompt deep links currently support:

- `app=claude`
- `app=claude_desktop`

Old multi-app identifiers are no longer supported.

## Suggestions

- If Claude Code and Claude Desktop should use the same prompt, create one preset for each target
- If you often edit `CLAUDE.md` outside YKW Bridge, check the backfilled content before switching
- For long-lived team prompts, manage them together with export or deep link workflows
