# 3.3 Skills 管理

## 概要

Skills は再利用可能な能力拡張です。現在の版では **Claude ワークフロー** 向けの Skills 管理だけを残しています。

YKW Bridge は Skills を単一の事実源 (SSOT) で管理します:

- デフォルト SSOT: `~/.ykw-bridge/skills/`
- バックアップディレクトリ: `~/.ykw-bridge/skill-backups/`

## Skills ページを開く

トップナビゲーションの **Skills** をクリックします。

## インストール可能な Skills を見つける

YKW Bridge は、プリセットリポジトリ、自分の GitHub リポジトリ、公開 `skills.sh` カタログから skills を検出できます。

主な操作:

1. 名前または説明で検索
2. インストール済み / 未インストールで絞り込み
3. **更新** を押してリモートリポジトリを再スキャン

## Skills をインストールする

1. 対象 skill を見つける
2. **インストール** をクリック
3. 完了まで待つ

インストール後、skill は SSOT に書き込まれ、Claude ワークフローディレクトリへ同期されます。

## アンインストールと復元

YKW Bridge は skill を削除する前にバックアップを作成します。

アンインストール後:

1. skill が SSOT から削除される
2. Claude ワークフローディレクトリの同期結果が削除される
3. インストール記録がデータベースから削除される

復元したい場合:

1. **バックアップから復元** を開く
2. バックアップ項目を選ぶ
3. 復元して再度有効化する

## リポジトリ管理

カスタムリポジトリには次を指定できます:

- Owner
- Repository
- Branch
- Subdirectory（任意）

リポジトリを削除しても、今後の更新が止まるだけで、すでにインストール済みの skills は自動削除されません。

## ZIP インストールとインポート

オンラインリポジトリ以外に次も対応しています:

- ZIP ファイルからのインストール
- 未管理ローカル skills のスキャン
- 既存 Claude 側 skill ディレクトリの取り込み

## 保存場所の切り替え

skill のソース保存場所は次の 2 種類の間で切り替えられます:

| 場所 | 説明 |
| --- | --- |
| `~/.ykw-bridge/skills/` | YKW Bridge 管理のデフォルト保存先 |
| `~/.agents/skills/` | 複数ツール共有向けのコミュニティ慣習ディレクトリ |

保存場所を切り替えても現在のインストール状態は失われません。

## 注意事項

- 現在の版では Claude 側 skill 同期のみ残っています
- 同期方式（symlink / copy）とソース保存場所は別設定です
- リモートリポジトリに接続できなくても、すでにインストール済みの skills は引き続き使えます
