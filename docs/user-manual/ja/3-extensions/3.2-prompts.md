# 3.2 Prompts 管理

## 概要

Prompts は再利用可能なシステムプロンプトプリセットを管理します。YKW Bridge は現在、次の 2 つの対象に対して個別のプロンプトセットを管理します:

- **Claude Code**
- **Claude Desktop**

この 2 つの対象は互いに独立しています。

## パネルを開く

トップナビゲーションの **Prompts** をクリックします。

## プリセットを作成する

1. **+** をクリック
2. プリセット名を入力
3. Markdown エディタでプロンプトを書く
4. **保存** をクリック

エディタの主な機能:

- Markdown シンタックスハイライト
- ライブプレビュー
- よく使う書式ショートカット

## プリセットを有効化する

カード上のトグルでプリセットを有効化します。

同じ対象では常に 1 つだけ有効です。新しいプリセットを有効にすると、以前のプリセットは自動で無効になります。

## 同期先

有効化後、YKW Bridge は内容を対応する live ファイルへ書き戻します:

| 対象 | 同期ファイル |
| --- | --- |
| Claude Code | `~/.claude/CLAUDE.md` |
| Claude Desktop | Claude Desktop profile ディレクトリ内の `CLAUDE.md` |

> Claude Desktop の profile パスはプラットフォームや設定のカスタムパスに依存します。アプリ内の status / Doctor 表示を基準にしてください。

## 編集と削除

### 編集

1. **編集** をクリック
2. 名前または内容を変更
3. **保存** をクリック

現在有効なプリセットを編集した場合、保存後すぐに live ファイルへ同期されます。

### 削除

1. **削除** をクリック
2. 操作を確認

現在有効なプリセットは直接削除できません。先に無効化するか、別のプリセットに切り替えてください。

## スマートバックフィル

プリセット切替前に、YKW Bridge は現在の live ファイルを読み込み、外部で手動編集した内容を現在のプリセットへバックフィルして、内容が失われないようにします。

よくある発動タイミング:

- 別のプリセットへ切り替えるとき
- 現在有効なプリセットを編集するとき
- 初回起動時に既存の live ファイルを取り込むとき

## Deep Link

Prompt の Deep Link は現在次をサポートします:

- `app=claude`
- `app=claude_desktop`

古い複数アプリ識別子はもうサポートされません。

## 使い方のヒント

- Claude Code と Claude Desktop で同じ内容を使いたい場合でも、対象ごとに別プリセットを作ってください
- 外部で `CLAUDE.md` をよく手編集する場合は、切替前にバックフィル内容を確認すると安全です
- 長期運用のチーム用プロンプトは export や deep link と組み合わせて管理すると扱いやすいです
