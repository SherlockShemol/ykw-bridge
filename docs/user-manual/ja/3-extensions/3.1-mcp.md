# 3.1 MCP サーバー管理

## 概要

YKW Bridge は現在、**Claude Code** 向けの MCP サーバーのみを管理します。

MCP 設定は YKW Bridge のデータベースに保存され、有効化すると `~/.claude.json` の `mcpServers` に同期されます。現在の UI には Claude ワークフロー用の MCP トグルだけが残っています。

## MCP パネルを開く

トップナビゲーションの **MCP** をクリックします。

## MCP サーバーを追加する

### プリセットテンプレートを使う

1. 右上の **+** をクリック
2. プリセットのドロップダウンからテンプレートを選択
3. 必要に応じてコマンド、引数、URL を調整
4. **保存** をクリック

よく使うプリセット:

- `fetch`
- `time`
- `memory`
- `sequential-thinking`
- `context7`

### カスタム設定

カスタムモードでよく使う項目:

| 項目 | 必須 | 説明 |
| --- | --- | --- |
| Server ID | はい | 一意の識別子 |
| Name | いいえ | 表示名 |
| Description | いいえ | 説明 |
| Transport Type | はい | `stdio` / `http` / `sse` |
| Command | `stdio` では必須 | 起動コマンド |
| Arguments | いいえ | コマンド引数 |
| URL | `http` / `sse` では必須 | リモートサービス URL |
| Headers | いいえ | HTTP ヘッダー |
| Environment Variables | いいえ | MCP プロセスに渡す環境変数 |

## 有効化と同期

MCP パネルには現在 **Claude** トグルのみがあります。

有効化すると:

1. データベースに Claude 向け有効状態が保存される
2. 設定が `~/.claude.json` に書き込まれる
3. Claude Code が次回起動時または設定再読込時に読み込む

無効化すると:

1. データベース状態が更新される
2. 対象サーバーが `~/.claude.json` から削除される

## 既存設定のインポート

すでに Claude 側で MCP サーバーを設定している場合は **インポート** を使えます:

1. 現在の `~/.claude.json` を読み取る
2. その `mcpServers` を YKW Bridge に取り込む
3. 以後の変更を YKW Bridge 側で一元管理する

## 設定例

### `stdio`

```json
{
  "command": "uvx",
  "args": ["mcp-server-fetch"]
}
```

### `http`

```json
{
  "url": "http://localhost:8080/mcp"
}
```

### Claude MCP ファイル例

```json
{
  "mcpServers": {
    "mcp-fetch": {
      "command": "uvx",
      "args": ["mcp-server-fetch"]
    }
  }
}
```

## Deep Link インポート

MCP の Deep Link は現在次のみ対応します:

- `resource=mcp`
- `apps=claude`

古い複数アプリ識別子を含むリンクはインポートに失敗します。

## 注意事項

- `stdio` モードでは、コマンドがすでにインストールされ `PATH` に入っている必要があります
- MCP を変更した後は Claude Code を再度開くか、新しいセッションを開始するのが安全です
- ローカルの Claude 設定パスが存在しない場合、live ファイルには書き込まれないことがありますが、データベース設定は保持されます

## よくある問題

### サーバーが起動しない

まず確認すること:

- コマンドがインストールされているか
- 引数が正しいか
- 必要なローカル環境変数が揃っているか

### 設定が反映されない

まず確認すること:

- Claude トグルが有効か
- `~/.claude.json` が他のツールに上書きされていないか
- Claude Code が設定を再読込したか
