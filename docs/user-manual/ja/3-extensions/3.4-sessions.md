# 3.4 セッションマネージャー

## 概要

現在のセッションマネージャーは **Claude Code** のローカルセッションログのみを閲覧・管理します。

セッションファイルは Claude のプロジェクトセッションディレクトリから取得されます:

- `~/.claude/projects/` 配下の JSONL セッションファイル

古い他ワークフロー向けのセッション入口は現在の版から削除されています。

## セッションマネージャーを開く

トップナビゲーションの **Sessions** をクリックします。

## できること

- 最近のセッションを閲覧
- タイトル、ディレクトリ、概要、セッション ID で検索
- 完全なメッセージ履歴を表示
- 再開コマンドをコピー
- 単一または複数のローカルセッションファイルを削除

## 画面構成

### 左側リスト

各セッションに表示される内容:

- タイトル
- 最終アクティブ時刻
- プロジェクトディレクトリ

### 右側詳細

選択すると次を確認できます:

- 完全なタイムスタンプ
- プロジェクトディレクトリ
- 再開コマンド
- 全メッセージ履歴

## 検索とフィルター

現在のフィルターは次に整理されています:

- **All**
- **Claude**

`Esc` を押すと検索ボックスをすばやくクリアできます。

## セッション再開

**Resume** をクリックすると:

- macOS では YKW Bridge が直接ターミナル起動を試みる
- 他プラットフォームでは再開コマンドのコピーを優先する

ターミナル起動に失敗した場合は、コマンドコピーにフォールバックします。

## セッション削除

### 単一セッション削除

詳細パネルの削除操作を使うと、確認後に対応するローカルセッションファイルが削除されます。

### 一括削除

1. 一括モードに入る
2. 複数セッションを選択
3. 一括削除を実行

## ディレクトリピッカー

現在のディレクトリピッカーは **Claude セッション再開** をより安定させるためのものです。古いワークフローの再開パスは残っていません。

## 注意事項

- セッションは最終アクティブ時刻の降順で並びます
- 書き込み可能なローカル Claude セッションファイルのみ削除できます
- ローカル元ファイルのない読み取り専用記録は削除できません
