# 5.1 設定ファイル

## YKW Bridge データディレクトリ

### デフォルト位置

現在のデフォルトデータディレクトリは `~/.ykw-bridge/` です。

設定で個別パスを上書きしていない限り、データベース、端末単位設定、Skills、Workspace、バックアップ、Claude Desktop managed profile はすべてここに置かれます。

### よくあるディレクトリ構成

```text
~/.ykw-bridge/
├── ykw-bridge.db              # メイン SQLite データベース
├── settings.json              # 端末単位設定
├── config.json                # 集約設定ファイル
├── backups/                   # DB / import / env 削除前バックアップ
├── skills/                    # Skills の単一事実源 (SSOT)
├── skill-backups/             # skill アンインストール時バックアップ
├── workspace/                 # Workspace ファイル
│   └── memory/                # Daily Memory
├── claude-desktop/
│   └── profile/               # Claude Desktop managed profile（デフォルト）
└── crash.log                  # 必要時に作成されるクラッシュログ
```

### データベースに入るもの

`ykw-bridge.db` は現在のメインデータベースで、主に次を保存します:

- provider 設定と custom endpoint
- MCP サーバー
- Prompt プリセット
- skill のインストール状態とリポジトリ情報
- proxy、failover、request log
- モデル料金と健康状態

### `settings.json`

`settings.json` は端末単位設定を保存し、データベースと一緒には端末間同期されません。代表的な内容:

```json
{
  "visibleApps": {
    "claude": true,
    "claudeDesktop": true
  },
  "claudeConfigDir": null,
  "claudeDesktopAppPath": null,
  "claudeDesktopProfileDir": null,
  "launchOnStartup": false
}
```

### `backups/`

`backups/` には主に次のバックアップが入ります:

- import 前に自動作成されるバックアップ
- database restore 前の安全バックアップ
- 競合環境変数削除前に作られる `env-backup-*.json`

## Claude Code 設定

### デフォルトディレクトリ

デフォルトパス: `~/.claude/`

### よくあるファイル

```text
~/.claude/
├── settings.json     # Claude Code メイン設定
├── CLAUDE.md         # Prompt / system instructions
└── skills/           # 同期済み skills
```

### MCP 設定ファイル

Claude Code の MCP 設定はデフォルトで `~/.claude.json` に書き込まれます。

設定で Claude Code の config ディレクトリを上書きした場合、MCP ファイルもそれに追従し、固定のデフォルトパスではなく、隣接する `*.json` sidecar ファイルへ書き込まれます。

## Claude Desktop Managed Profile

### デフォルトディレクトリ

デフォルトパス: `~/.ykw-bridge/claude-desktop/profile/`

設定でこのパスを変更することもできます。

### ここに入る主なもの

managed profile には Claude Desktop ワークフローに必要なファイルが入ります。例:

- managed config
- `CLAUDE.md`
- `skills/`
- ローカル gateway / 証明書関連ファイル
- Claude Desktop のローカルセッションディレクトリ

これは Claude Desktop ワークフロー用の managed directory であり、単独の第三者アプリ設定ディレクトリではありません。

## Workspace と Daily Memory

デフォルトパス:

- `~/.ykw-bridge/workspace/`
- `~/.ykw-bridge/workspace/memory/`

Workspace パネルのファイルと Daily Memory はここに保存されます。

## 設定の優先層

現在の YKW Bridge の挙動は大きく 3 層で考えられます:

1. **Database**: 主データソース
2. **Claude / Claude Desktop の live ファイル**: provider、prompt、MCP、skills 変更時に書き込まれる
3. **Backfill 読み取り**: 現在設定を編集するとき、既存 live ファイルを UI に読み戻そうとする

## 手動編集ガイド

### 手動編集してよいもの

- Claude Code の設定ファイル
- 用途を理解している Claude Desktop managed profile 内のプレーンテキストファイル
- YKW Bridge の `settings.json`

### 手動編集を勧めないもの

- `ykw-bridge.db`
- `backups/` 内のファイル
- 用途を十分理解していない Claude Desktop managed gateway ファイル

### 手動編集後の再同期

Claude 関連の live 設定を手で変更した場合:

1. YKW Bridge を開く
2. 関連 provider または設定パネルへ移動
3. バックフィル内容が正しいか確認
4. もう一度保存して database と live ファイルを揃える

## 設定移行

### 旧 `.cc-switch`

古いバージョンでは `~/.cc-switch/` を使っていた場合があります。現在のデフォルトディレクトリは `~/.ykw-bridge/` です。

新ディレクトリが存在せず旧ディレクトリが存在する場合、YKW Bridge は互換のために旧データを自動インポートすることがあります。実務上は:

- `~/.cc-switch/` は旧互換の入力元
- `~/.ykw-bridge/` は現在のデフォルト位置

### デバイス間移行

移行方法は 3 つあります:

1. 設定を export / import する
2. WebDAV sync を使う
3. `~/.ykw-bridge/` の必要部分を手動でバックアップして復元する

## バックアップのすすめ

### 定期 export

設定内のデータ管理パネルから定期的に設定を export するのがおすすめです。特に:

- 大規模な provider 変更前
- Prompt / MCP / Skills の一括変更前
- デバイス切替前

### export に通常含まれるもの

- provider 設定
- MCP サーバー
- Prompt プリセット
- skill metadata
- 同期向きのアプリデータ

### これだけに頼らないほうがよいもの

runtime log、temporary state file、device-specific path override だけを、端末間復旧の唯一の根拠にしないでください。
