# 5.3 Deep Link プロトコル

## 概要

YKW Bridge は現在 `ykwbridge://` deep link プロトコルを使い、provider、prompt、MCP 設定、skill repository をワンクリックで import できます。

**よくある用途**:

- team で provider や MCP 設定を共有する
- tutorial で one-click import を提供する
- 複数デバイスで同じ構成を再利用する

## プロトコル形式

### V1 プロトコル

```text
ykwbridge://v1/import?resource={type}&...
```

### 共通パラメータ

| パラメータ | 必須 | 説明 |
|------|------|------|
| `resource` | はい | `provider` / `prompt` / `mcp` / `skill` |

## リソース種別ごとのパラメータ

### Provider (`resource=provider`)

| パラメータ | 必須 | 説明 |
|------|------|------|
| `app` | はい | `claude` / `claude_desktop` |
| `name` | はい | provider 名 |
| `endpoint` | いいえ | API endpoint。カンマ区切りで複数 URL 対応 |
| `apiKey` | いいえ | API key |
| `homepage` | いいえ | homepage URL |
| `model` | いいえ | デフォルトモデル |
| `haikuModel` | いいえ | Claude Haiku デフォルトモデル |
| `sonnetModel` | いいえ | Claude Sonnet デフォルトモデル |
| `opusModel` | いいえ | Claude Opus デフォルトモデル |
| `notes` | いいえ | メモ |
| `icon` | いいえ | icon 名 |
| `config` | いいえ | Base64 エンコード済み inline config |
| `configFormat` | いいえ | `json` / `toml` |
| `enabled` | いいえ | import 後すぐ有効にするか |
| `usageScript` | いいえ | Base64 エンコード済み usage query script |
| `usageEnabled` | いいえ | usage query を有効にするか |
| `usageApiKey` | いいえ | usage query 用 API key |
| `usageBaseUrl` | いいえ | usage query 用 base URL |
| `usageAccessToken` | いいえ | usage query 用 access token |
| `usageUserId` | いいえ | usage query 用 user ID |
| `usageAutoInterval` | いいえ | 自動問い合わせ間隔（分） |

補足:

- `endpoint`、`apiKey`、`homepage` は URL に直接書けます
- `config` に Base64 inline config を渡して、YKW Bridge に値をバックフィルさせることもできます
- この manual では remote `configUrl` を主要ルートとしてはもう推奨しません

### Prompt (`resource=prompt`)

| パラメータ | 必須 | 説明 |
|------|------|------|
| `app` | はい | `claude` / `claude_desktop` |
| `name` | はい | prompt 名 |
| `content` | はい | Base64 エンコード済み prompt 内容 |
| `description` | いいえ | 説明 |
| `enabled` | いいえ | import 後すぐ有効にするか |

### MCP (`resource=mcp`)

| パラメータ | 必須 | 説明 |
|------|------|------|
| `apps` | はい | 現在は `claude` のみ対応 |
| `config` | はい | `mcpServers` を含む Base64 エンコード済み JSON |
| `enabled` | いいえ | 有効化するか |

### Skill Repository (`resource=skill`)

| パラメータ | 必須 | 説明 |
|------|------|------|
| `repo` | はい | `owner/name` 形式の repository |
| `directory` | いいえ | repo 内ディレクトリ |
| `branch` | いいえ | branch 名 |

skill deep link には `app` は不要です。現在は Claude Skills repository 管理へ取り込まれます。

## import 例

### Claude provider を import

```text
ykwbridge://v1/import?resource=provider&app=claude&name=My%20Provider&endpoint=https%3A%2F%2Fapi.example.com&apiKey=sk-xxx
```

### Claude Desktop provider を import

```text
ykwbridge://v1/import?resource=provider&app=claude_desktop&name=Desktop%20Provider&endpoint=https%3A%2F%2Fapi.example.com&apiKey=sk-xxx
```

### MCP を import

```text
ykwbridge://v1/import?resource=mcp&apps=claude&config=BASE64_ENCODED_JSON
```

### Prompt を import

```text
ykwbridge://v1/import?resource=prompt&app=claude&name=Code%20Review&content=BASE64_ENCODED_CONTENT
```

### Skill repository を import

```text
ykwbridge://v1/import?resource=skill&repo=owner/repo&directory=skills/my-skill&branch=main
```

## リンク生成方法

### 手動生成

1. パラメータを準備
2. `ykwbridge://v1/import?...` 形式の URL を組み立てる
3. 特殊文字を URL エンコードする
4. `content`、`config`、`usageScript` などは Base64 エンコードする

### JavaScript 例

```javascript
const params = new URLSearchParams({
  resource: "provider",
  app: "claude",
  name: "My Provider",
  endpoint: "https://api.example.com",
  apiKey: "sk-xxx",
});

const url = `ykwbridge://v1/import?${params.toString()}`;
```

### Base64 例

```javascript
const promptContent = Buffer.from("# Role\nYou are a reviewer", "utf8").toString("base64");
```

## Deep Link を使う

deep link をクリックすると、YKW Bridge は:

1. プロトコルとパラメータを検証
2. import confirmation dialog を開く
3. 設定プレビューを表示
4. 確認後に import を実行

**Security tip**: 信頼できる送信元のリンクだけを import してください。

## プロトコル登録

### 自動登録

YKW Bridge はインストール時に `ykwbridge://` プロトコルを自動登録します。

### 手動確認

プロトコルが動作しない場合:

- macOS: アプリを再インストールするか登録フローを再実行
- Windows: `HKEY_CLASSES_ROOT\ykwbridge` を確認
- Linux: `.desktop` ファイル内のプロトコル宣言を確認

## セキュリティ上の提案

deep link には次のような機密情報が入る場合があります:

- API key
- usage query script
- third-party endpoint

提案:

1. 実 credential を含むリンクを公開共有しない
2. 共有前に機密値を置き換える
3. import 前に preview dialog を確認する

## トラブルシューティング

### リンクが開かない

まず確認:

1. YKW Bridge がインストールされているか
2. `ykwbridge://` が登録されているか
3. リンク形式が正しいか

### import に失敗する

**よくある原因**:

- Base64 エンコード不正
- JSON / TOML 内容不正
- 必須項目不足
- `app` / `apps` に古い非対応値が残っている

**対処**:

1. 元データを再確認
2. Base64 項目を再エンコード
3. `provider` / `prompt` は `claude` または `claude_desktop` のみ使う
4. `mcp.apps` は `claude` のみ使う
