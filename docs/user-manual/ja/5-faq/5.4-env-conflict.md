# 5.4 環境変数競合

## 概要

YKW Bridge は、system environment variable と現在の Claude ワークフロー設定の競合を自動検出し、設定が意図せず上書きされるのを防ぎます。

この競合チェックは現在 `Claude` / `Claude Desktop` のみを対象にします。manual では古い複数アプリ用 environment variable フローはもう扱いません。

**現在の走査対象**:

- `ANTHROPIC_API_KEY`
- `ANTHROPIC_AUTH_TOKEN`
- `ANTHROPIC_BASE_URL`
- 名前に `ANTHROPIC` を含むその他の変数

## 競合警告

競合が検出されると、UI 上部に黄色い warning banner が表示されます:

```text
⚠️ Environment variable conflicts detected
Found X environment variables that may conflict with YKW Bridge config
[Expand] [Close]
```

## 競合詳細を見る

**Expand** を押すと次を確認できます:

| 項目 | 説明 |
|------|------|
| Variable Name | 環境変数名 |
| Variable Value | 現在の値 |
| Source | どこから来たか |

### Source 種別

| Source | 説明 |
|------|------|
| User Registry | Windows の user-level environment variables |
| System Registry | Windows の system-level environment variables |
| Shell Config | macOS/Linux の shell config files |
| Process Environment | 現在プロセスの environment variables |

macOS / Linux では、現在の process environment に加えて、次のような shell config files も確認します:

- `~/.bashrc`
- `~/.bash_profile`
- `~/.zshrc`
- `~/.zprofile`
- `~/.profile`
- `/etc/profile`
- `/etc/bashrc`

## 競合の処理

### 削除する変数を選ぶ

1. 削除したい環境変数を選択
2. または **Select All** で全部選択

### 変数を削除する

1. **Delete Selected** を押す
2. 操作を確認する
3. YKW Bridge は先に backup を作成し、その後で選択した変数を削除する

### 自動バックアップ

削除前に JSON backup が作成されます:

- backup 位置: `~/.ykw-bridge/backups/`
- 例: `env-backup-20260418_120000.json`
- backup 内容: 変数名、値、source location

## 警告を閉じる

これらの変数が現在の設定に影響しないと判断できる場合:

1. banner 右側の **Close** を押す
2. 現在セッション中は警告が隠れる
3. 次回起動時に再検出される

## 手動対応

YKW Bridge に削除させたくない場合は手動でも処理できます。

### Windows

1. **System Properties → Advanced → Environment Variables** を開く
2. user 変数または system 変数から対象を探す
3. 削除または修正する

### macOS / Linux

1. `~/.zshrc` や `~/.bashrc` などの shell config を編集
2. 対応する `export` 行を削除またはコメントアウト
3. たとえば `source ~/.zshrc` で再読込

## 競合が問題になる理由

環境変数は通常 config file より優先度が高く、次のような問題を起こします:

- YKW Bridge が書いた provider 設定が上書きされる
- リクエストが誤った endpoint に送られる
- 誤った認証情報が使われる

## ベストプラクティス

1. Claude 関連設定はできるだけ YKW Bridge 内で管理する
2. shell config を整理し、重複する `ANTHROPIC_*` 変数を何度も export しない
3. 削除前に backup が作成されたことを確認する

## 削除した変数を戻す

誤って環境変数を削除した場合:

1. `~/.ykw-bridge/backups/` を開く
2. 対応する `env-backup-*.json` を探す
3. backup 内容をもとに手動で復元する
