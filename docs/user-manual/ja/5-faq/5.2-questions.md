# 5.2 よくある質問

## インストール

### macOS インストール

YKW Bridge の macOS ビルドは署名と notarization 済みです。通常はそのままインストールできます。問題がある場合は、まず [Releases ページ](https://github.com/SherlockShemol/ykw-bridge/releases) から最新版を再ダウンロードしてください。

### Windows で起動しない

**よくある原因**:

- WebView2 runtime が不足している
- アンチウイルスにブロックされている

**対処**:

1. [Microsoft Edge WebView2](https://developer.microsoft.com/en-us/microsoft-edge/webview2/) をインストール
2. YKW Bridge をアンチウイルスの許可リストへ追加

### Linux 起動エラー

**問題**: AppImage が起動しない

**対処**:

```bash
chmod +x YKW-Bridge-*.AppImage
./YKW-Bridge-*.AppImage --no-sandbox
```

## Provider 問題

### provider を切り替えても反映されない

**理由**: ワークフローごとに設定を読み込むタイミングが違います。

**対処**:

- Claude Code 通常モード: ターミナルを閉じて再度開く、または IDE 内の Claude Code プロセスを再起動
- Claude Code takeover / proxy モード: 多くの場合そのままホット反映される
- Claude Desktop managed profile: 現在のアプリインスタンスが古い設定を保持している場合、Claude.app を終了して再度開く

### API Key が無効

**確認**:

1. 余計な空白や改行が入っていないか
2. 期限切れや無効化が起きていないか
3. endpoint と認証方式が一致しているか
4. speed test や stream check で実際の接続性を確認

### 公式ログインへ戻す方法

**手順**:

1. Claude の **Official Login** プリセットを選ぶ
2. **Enable** を押す
3. 対応する Claude ワークフローを再度開く
4. 公式ログインフローを完了する

> `codex_oauth` は引き続き残っていますが、Claude ワークフローへ接続する provider 経路であり、単独の Codex アプリではありません。

## Proxy 問題

### ルーティングサービスが起動しない

**よくある原因**: ポートが使用中

**対処**:

1. ポート使用状況を確認:

   ```bash
   # macOS/Linux
   lsof -i :49152

   # Windows
   netstat -ano | findstr :49152
   ```

2. そのポートを使っているプロセスを停止
3. または **設定 → ルーティングサービス** でデフォルトポートに戻す

### proxy モードでタイムアウトする

**よくある原因**:

- ネットワーク不安定
- upstream provider 障害
- proxy 設定ミス

**対処**:

1. まずローカルネットワーク疎通を確認
2. 一時的に proxy を無効化して provider へ直接接続し比較
3. provider カードの endpoint、認証、モデル設定を確認

### proxy 停止後に設定が戻らない

**よくある原因**: 復元処理が完了する前に proxy が異常終了した

**対処**:

1. 現在の provider を開く
2. endpoint と認証項目を確認
3. もう一度保存して live 設定を現在の database 状態に戻す

## フェイルオーバー問題

### なぜフェイルオーバーしないのですか？

**チェックリスト**:

- [ ] ルーティングサービスが動作中
- [ ] app takeover が有効
- [ ] 自動フェイルオーバーが有効
- [ ] キューに代替 provider が本当に入っている

### なぜ頻繁に切り替わるのですか？

**よくある原因**:

- primary provider が不安定
- breaker しきい値が厳しすぎる

**対処**:

1. まず primary provider を確認
2. failure threshold を少し緩める
3. 必要なら primary provider を入れ替える

### 全 provider が tripped の場合は？

**対処**:

1. breaker 回復待機時間が終わるまで待つ
2. またはルーティングサービスを再起動して状態をリセットする

## データ問題

### 設定が消えたように見える

**よくある原因**:

- データディレクトリが削除された
- database が壊れている
- まだ古い `.cc-switch` 位置を見ている

**対処**:

1. `~/.ykw-bridge/` が存在するか確認
2. `~/.ykw-bridge/backups/` に復元可能なバックアップがあるか確認
3. 必要なら export した設定ファイルから再 import

### import に失敗する

**よくある原因**:

- ファイル形式が不正
- 内容が不完全
- データバージョン非互換

**対処**:

1. そのファイルが YKW Bridge による export か確認
2. テキストエディタで JSON が完全か確認
3. より現行版に近いバックアップを使って再 export / 再試行する

### 使用量統計が空

**チェックリスト**:

- [ ] ルーティングサービスが動作中
- [ ] app takeover が有効
- [ ] logging が有効
- [ ] 実際のリクエストが現在の経路を通っている

## Quota と Balance

### なぜ自動表示される provider と手動設定が必要な provider があるのですか？

現在の manual で自動 quota query 対象として残しているのは:

- Claude official login
- GitHub Copilot
- Codex OAuth reverse proxy

それ以外の provider、たとえば Token Plan、third-party balance query、relay service などは、provider カードの **Usage Query** パネルで手動有効化が必要です。同じ request URL が plan quota と account balance の両方を提供できるため、YKW Bridge は自動では決め打ちしません。

### 公式サブスクリプション系 provider に quota が出ない

**確認**:

1. 現在有効な provider になっているか
2. Copilot / Codex OAuth なら OAuth token がまだ有効か
3. カードに **Session Expired** が出ていれば **OAuth Auth Center** で再ログイン
4. カードの refresh ボタンで再問い合わせ

### 手動有効化したのに使用量が出ない

**確認**:

1. **Enable Usage Query** が有効か
2. 正しいテンプレートを選んで保存したか
3. **Test Script** で実際のエラーを確認したか
4. provider が現在アクティブか

## Codex OAuth Reverse Proxy

### Codex OAuth Reverse Proxy を有効化するリスクは？

Codex OAuth reverse proxy は reverse-engineered OAuth flow を使って ChatGPT アカウント経由で Codex service にアクセスします。OpenAI の Terms of Service に抵触する可能性があり、アカウント制限、挙動変化、長期的不安定性のリスクがあります。有効化前に自分で判断してください。

完全な免責事項は [2.1 プロバイダーの追加 → Codex OAuth Reverse Proxy](../2-providers/2.1-add.md#codex-oauth-リバースプロキシclaude-プロバイダー) を参照してください。

### Codex OAuth へのログイン方法

完全な Device Code ログインフロー、2 つの入口（Add Provider panel / OAuth Auth Center）、マルチアカウント管理、よくある失敗ケースは [2.1 プロバイダーの追加 → Codex OAuth Reverse Proxy (Claude Provider)](../2-providers/2.1-add.md#codex-oauth-リバースプロキシclaude-プロバイダー) を参照してください。

### Codex OAuth にログイン済みなのに quota が表示されない

**対処**:

1. **OAuth Auth Center** でログイン完了済みか確認
2. token がまだ有効か確認
3. カードに **Session Expired** が出ていれば、そのアカウントを削除して再ログイン

## その他

### tray icon が表示されない

**macOS**:

- menu bar icon の表示設定を確認

**Windows**:

- taskbar の hidden area に折りたたまれていないか確認

**Linux**:

- `libappindicator` など system tray support package が入っているか確認

### UI 表示が崩れる

**対処**:

1. theme を切り替えて再試行
2. アプリ再起動
3. `~/.ykw-bridge/settings.json` を削除して端末単位設定をリセット

### update に失敗する

**対処**:

1. ネットワーク接続を確認
2. Releases ページから最新版を再ダウンロードして上書きインストール

## Lightweight Mode

### Lightweight Mode へ入る方法

system tray menu から **Lightweight Mode** を切り替えます。メインウィンドウは閉じますが、YKW Bridge は tray に残って動作します。もう一度切り替えるか **Open Main Window** を押すと戻れます。

### Lightweight Mode でメモリ使用量は下がりますか？

はい。Lightweight Mode では main window と WebView を破棄するため、通常はメモリ使用量が目に見えて減ります。

### Lightweight Mode 中でも deep link でメイン画面を開けますか？

はい。`ykwbridge://` リンクをクリックすると、YKW Bridge は必要に応じて main window を再構築し、import confirmation dialog を開きます。初回は少し遅く感じることがありますが、その後は通常に戻ります。

## ヘルプを得る

### Issue を作成する

上記の方法でも解決しない場合:

1. [GitHub Issues](https://github.com/SherlockShemol/ykw-bridge/issues) を開く
2. 似た issue があるか検索
3. 必要なら新規 issue を作成
4. 次を含める:
   - OS とバージョン
   - YKW Bridge バージョン
   - 再現手順
   - 関連エラー

### あると助かる診断情報

合わせて次のような関連情報を添えると助かります:

- `~/.ykw-bridge/` 配下の crash log や backup file
- provider カードのエラー画面
- proxy や stream check のエラー出力
